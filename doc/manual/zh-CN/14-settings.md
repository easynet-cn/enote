## 14. 应用设置

点击侧边栏底部的"设置"图标按钮打开设置对话框，可配置以下选项。

### 14.1 主题设置

ENote 支持三种主题模式：

- **浅色模式：** 经典的白色背景界面，适合日间使用。
- **深色模式：** 深色背景界面，减少眼部疲劳，适合夜间或低光环境使用。
- **跟随系统：** 自动跟随操作系统的深浅色设置，系统切换主题时应用界面实时跟随变化。

在设置对话框的"外观"区域，点击对应的主题按钮即可切换。主题切换即时生效，设置自动保存。

### 14.2 语言设置

ENote 支持中文简体（zh-CN）和英文（en-US）两种界面语言。

在设置对话框的"外观"区域，点击对应的语言按钮即可切换。切换后界面所有文本立即更新，无需重启应用。语言偏好设置会自动保存，下次启动时自动应用。

### 14.3 布局模式设置

ENote 支持根据窗口宽度自动适配布局，也可手动指定布局模式：

| 模式 | 说明 |
|------|------|
| **自动（默认）** | 根据窗口宽度自动选择：> 1024px 桌面模式，640-1024px 平板模式，< 640px 手机模式 |
| **桌面** | 强制使用三栏并排布局 |
| **平板** | 强制使用双栏布局（侧边栏覆盖层） |
| **手机** | 强制使用单视图全屏切换布局 |

在设置对话框的"外观"区域，点击对应的布局按钮即可切换。也可使用快捷键 Ctrl+Shift+M（macOS 为 Cmd+Shift+M）循环切换，或通过命令面板搜索"切换布局"执行。

### 14.4 快捷键设置

在设置对话框的"快捷键"区域，可自定义应用级键盘快捷键。

**系统级快捷键**（如 Ctrl+S 保存、Esc 取消编辑）为固定绑定，无法修改，以"系统"标签标识。

**可自定义快捷键**的操作方式：

1. 点击快捷键显示区域，进入录制状态，界面提示"请按下快捷键..."。
2. 按下新的快捷键组合（如 Ctrl+K），录制完成后自动保存。
3. 如果新快捷键与已有快捷键冲突，显示红色冲突提示，不会保存。
4. 按 Esc 取消录制。

每个自定义快捷键右侧有「重置」按钮，可恢复为默认值。顶部有「全部重置为默认」按钮，一键恢复所有快捷键。

快捷键设置自动持久化，重启应用后仍然有效。命令面板中显示的快捷键文本会同步更新。

### 14.5 自动备份设置

在设置对话框的"自动备份"区域，可配置以下选项：

| 选项 | 说明 |
|------|------|
| **启用自动备份** | 开关控件，开启或关闭自动备份功能 |
| **备份间隔** | 下拉选择备份频率：1小时、4小时、8小时、24小时 |
| **保留份数** | 下拉选择保留的最大备份数量：最近5份、10份、20份、50份 |
| **立即备份** | 手动触发一次备份操作 |
| **上次备份** | 显示最近一次自动备份的文件名和时间 |

启用自动备份后，系统在后台按设定间隔自动执行备份，并根据保留份数自动清理旧备份文件。

### 14.6 安全设置（锁屏）

ENote 提供锁屏安全功能，可在应用启动时或一段时间无操作后自动锁定界面，需要输入密码方可继续使用，防止未经授权的访问。

**默认状态：** 锁屏功能默认关闭，用户可根据需要手动开启。

在设置对话框的"安全"区域，可配置以下选项：

| 选项 | 说明 |
|------|------|
| **锁屏模式** | 切换锁屏保护的开启/关闭状态。可选"关闭"或"密码" |
| **设置密码** | 首次开启时设置锁屏密码（至少 4 位） |
| **修改密码** | 已设置密码时，可输入新密码替换当前密码 |
| **移除密码** | 清除已设置的锁屏密码并关闭锁屏功能 |
| **超时锁定** | 设置无操作自动锁定的等待时间：仅启动时、1分钟、5分钟、15分钟、30分钟 |
| **最小化时锁定** | 开关控件，开启后应用窗口最小化或切换到后台时自动锁定 |

**锁屏界面：**

当应用被锁定时，显示全屏半透明遮罩层和居中的锁屏卡片。用户需输入正确的锁屏密码并点击"解锁"按钮方可进入应用。密码错误时显示错误提示并播放抖动动画。

**手动锁定：** 按 Ctrl+L（macOS 为 Cmd+L）快捷键可随时手动锁定应用，也可通过命令面板搜索"锁定应用"执行。

**密码安全：** 锁屏密码使用 Argon2id 算法进行哈希处理后存储，不以明文形式保存，确保密码安全。

### 14.7 MCP 设置

MCP（Model Context Protocol）是一项 AI 工具集成功能，允许 Claude Desktop、Claude Code 等 AI 客户端通过标准协议操作笔记数据。MCP 功能默认关闭，需在设置中手动启用。

在设置对话框的"MCP (AI 工具集成)"区域，可配置以下选项：

| 选项 | 说明 |
|------|------|
| **启用 MCP** | 总开关，控制是否允许 AI 工具通过 MCP 协议操作笔记 |
| **搜索笔记** | 允许 AI 工具搜索笔记内容 |
| **获取笔记详情** | 允许 AI 工具读取笔记完整内容 |
| **创建笔记** | 允许 AI 工具创建新笔记 |
| **更新笔记** | 允许 AI 工具修改已有笔记 |
| **删除笔记** | 允许 AI 工具删除笔记（移入回收站） |
| **列出笔记本** | 允许 AI 工具查看笔记本列表 |
| **创建笔记本** | 允许 AI 工具创建新笔记本 |
| **列出标签** | 允许 AI 工具查看标签列表 |
| **创建标签** | 允许 AI 工具创建新标签 |
| **笔记统计** | 允许 AI 工具获取笔记数量统计 |

启用总开关后，展开各工具的独立开关列表，可精细控制每个 MCP 工具的可用性。设置实时生效，无需重启 MCP Server。

> **注意：** MCP 设置控制 AI 工具的全局和工具级访问权限。禁用某个工具后，AI 客户端将无法看到和调用该工具。通过 AI 工具创建或修改的笔记，在历史记录中会标记为"AI 工具"来源，便于区分人工操作和 AI 操作。
>
> 除了全局和工具级控制外，ENote 还支持更细粒度的**数据级访问控制**——可以在笔记本、标签和单条笔记上分别设置 AI 的访问权限（继承/读写/只读/禁止），详见 [20.4 权限控制](20-mcp.md#204-权限控制)。

### 14.8 配置管理

在设置对话框中，**配置管理** 区块提供以下功能：

- **当前配置信息：** 显示当前使用的 Profile 名称和数据库类型。
- **切换配置：** 点击「切换配置」按钮，返回配置选择页面。在配置选择页面中可以：
  - 选择已有的数据库配置并连接
  - 点击「新建配置」进入设置向导，创建新的数据库配置
  - 编辑或删除已有配置
  - 点击右上角 ✕ 按钮关闭选择页面，返回主界面

### 14.9 系统维护（跨 Profile 同步）

在设置对话框底部的"系统维护"区域，提供跨 Profile 数据同步和同步记录管理功能。

#### 14.9.1 跨 Profile 同步

将当前 Profile 的数据同步到其他 Profile，支持跨数据库类型同步（SQLite ↔ MySQL ↔ PostgreSQL）。

**操作步骤：**

1. 在"系统维护"区域，点击「开始同步」按钮打开同步对话框。
2. 选择目标 Profile（下拉列表显示除当前 Profile 外的所有可用配置）。
3. 如果目标 Profile 使用密码认证（MySQL/PostgreSQL），需输入目标数据库密码。
4. 选择同步模式：

| 模式 | 说明 |
|------|------|
| **追加（默认）** | 将数据追加到目标数据库，不影响目标已有数据。ID 由目标库自增分配 |
| **覆盖** | 清空目标数据库后全量写入。选择此模式需二次确认 |

5. 勾选同步范围（可多选）：

| 数据类型 | 说明 |
|---------|------|
| **笔记本** | 笔记本及层级结构 |
| **标签** | 所有标签 |
| **笔记** | 笔记内容、标签关联、置顶状态、回收站状态 |
| **笔记历史** | 源端的原始历史记录 |
| **模板** | 笔记模板 |
| **设置** | 应用设置（键值对） |

6. 备份设置：建议开启「同步前自动备份」，可选择备份格式（SQL / Excel / CSV）。
7. 选择目标 Profile 后，下方显示源端和目标端的数据量预览。
8. 点击「开始同步」执行同步，界面显示实时进度（各表同步进度 + 总进度条）。

**关键行为：**

- **加密处理：** 如果源端和目标端的加密配置不同，系统自动处理——从源端解密读取，向目标端重新加密写入，确保数据一致性。
- **历史记录：** 笔记同步通过 service 层执行，目标端自动生成同步来源的历史记录（操作来源标记为"同步"）。
- **外键映射：** 笔记本和标签的 ID 在目标库中重新分配，笔记的笔记本归属和标签关联自动修正。

#### 14.9.2 同步记录管理

每次同步操作自动生成详细记录，包含每条数据的同步状态。

点击"系统维护"区域的「查看详情」按钮打开同步记录管理对话框。

**同步记录列表：** 显示所有历史同步操作，包含以下信息：

| 字段 | 说明 |
|------|------|
| 源 → 目标 | 源 Profile 和目标 Profile 名称 |
| 状态 | 完成 / 失败 |
| 成功/失败数 | 成功和失败的记录条数 |
| 同步模式 | 追加 / 覆盖 |
| 数据库类型 | 例如 sqlite → mysql |
| 时间 | 同步执行的时间 |

**操作：**

- **查看详情：** 展开单次同步的完整结果，包括汇总表格和逐条明细。明细支持按表名和状态筛选。
- **导出日志：** 将同步记录导出为 JSON 文件，便于排查问题。
- **删除记录：** 删除单条同步记录及其明细。
- **清空所有记录：** 清除全部历史同步记录。
