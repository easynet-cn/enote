## 13. 数据备份与恢复

ENote 提供完整的数据备份与恢复功能，支持手动导出和自动定时备份两种方式，确保数据安全。

点击侧边栏底部的"数据备份"图标按钮（数据库图标）打开数据备份对话框。对话框包含"导出备份"和"导入恢复"两个选项卡。

### 13.1 导出备份

**操作步骤：**

1. 打开数据备份对话框，默认处于"导出备份"选项卡。
2. 选择备份格式：
   - **SQL：** 导出为标准 SQL 语句文件（.sql），适合跨数据库迁移。
   - **Excel：** 导出为 Excel 表格（.xlsx），每张数据表对应一个 Sheet。
   - **CSV：** 导出为 CSV 文件，打包为 ZIP 压缩包（.zip）。
3. 点击"导出"按钮。
4. 在系统文件保存对话框中选择保存位置和文件名。
5. 导出完成后显示成功通知。

### 13.2 导入恢复

**操作步骤：**

1. 打开数据备份对话框，切换到"导入恢复"选项卡。
2. 选择与备份文件对应的格式（SQL / Excel / CSV）。
3. 点击"导入"按钮。
4. 在系统文件选择对话框中选择备份文件。
5. 系统弹出确认对话框，提示导入操作将清除当前所有数据并替换为备份数据。
6. 点击"确认导入"执行恢复，或点击"取消"放弃操作。
7. 恢复完成后显示成功通知。

> **注意：** 导入操作会覆盖当前所有数据，此操作不可撤销。执行导入前请确保已备份当前数据。导入完成后建议刷新页面以加载最新数据。

### 13.3 自动备份

ENote 支持自动定时备份功能，在后台按设定的时间间隔自动创建数据库备份，无需手动操作。

**工作原理：**

- 自动备份以 SQL 格式保存到应用数据目录下的 `backups/` 子目录中。
- 备份文件命名格式为 `enote_backup_YYYYMMDD_HHMMSS.sql`，包含完整的时间戳。
- 系统会根据设定的保留份数自动清理旧备份，避免占用过多磁盘空间。
- 应用启动时会自动检查是否需要执行备份（距上次备份是否超过设定间隔）。

**配置方式：** 在"设置"对话框的"自动备份"区域进行配置，详见 [14.5 自动备份设置](14-settings.md#145-自动备份设置)。

### 13.4 云备份

ENote 支持将备份文件上传到云存储，实现异地容灾。支持以下云存储提供商：

| 提供商 | 说明 |
|--------|------|
| **阿里云 OSS** | 阿里云对象存储服务，国内访问速度快 |
| **AWS S3** | 亚马逊云对象存储，全球覆盖广泛 |
| **腾讯云 COS** | 腾讯云对象存储，国内用户常用 |
| **MinIO** | S3 兼容的自建对象存储，适合私有部署 |
| **WebDAV** | 支持坚果云、NextCloud 等 WebDAV 协议的网盘服务 |

**工作原理：**

- 云备份基于本地自动备份的 SQL 文件——先在本地生成备份，再上传到云端。
- 云端备份文件存放在配置的目录前缀下（如 `enote-backups/`），文件名与本地一致。
- 系统根据设定的云端保留份数自动清理云端旧备份。
- 启用云备份后，每次自动备份完成时会自动上传到云端；云备份失败不影响本地备份。

**从云端恢复：**

1. 在云备份设置区域，点击"云端备份列表"查看所有云端备份文件。
2. 点击目标备份文件旁的"下载"按钮，将文件下载到本地 `backups/` 目录。
3. 通过"数据备份"对话框的"导入恢复"功能，选择下载的 SQL 文件进行恢复。

**安全说明：**

- 云存储的 Access Key Secret（或 WebDAV 密码）使用操作系统原生安全存储（macOS Keychain / Windows Credential Store / Linux Secret Service）保存，不以明文形式存储在数据库中。
- 所有云端传输均通过 HTTPS 加密。
- 如果启用了笔记内容加密，备份文件中的内容已经是密文状态。

**配置方式：** 在"设置"对话框的"云备份"区域进行配置，详见 [14.6 云备份设置](14-settings.md#146-云备份设置)。
