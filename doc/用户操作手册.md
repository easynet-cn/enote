## 1. 引言

### 1.1 编写目的

本手册旨在详细介绍 enote 智能笔记管理系统的各项功能和操作方法，帮助用户快速了解和掌握本软件的使用方式，充分利用软件提供的笔记管理能力，提高日常信息记录和知识管理的效率。

### 1.2 软件概述

enote 智能笔记管理系统是一款跨平台桌面笔记应用程序，为用户提供高效、便捷的笔记创建、编辑、管理和检索功能。软件支持富文本和 Markdown 两种编辑模式，具备笔记本分类管理、标签体系、全文搜索、历史版本追溯、多格式导入导出等核心功能，满足个人用户对知识管理和信息记录的全面需求。

软件采用前后端分离架构，前端基于 Vue 3 框架和 TypeScript 语言开发，后端采用 Rust 语言和 Tauri 2.x 框架，数据存储层使用 SeaORM 对象关系映射框架，默认支持 SQLite 数据库，同时兼容 MySQL 和 PostgreSQL 数据库。整体架构保证了软件的高性能、低资源占用和跨平台兼容性。

### 1.3 运行环境

**操作系统：**
- Windows 10 及以上版本（64位）
- macOS 10.15 (Catalina) 及以上版本
- Linux（主流发行版，如 Ubuntu 20.04+、Fedora 34+）

**硬件要求：**
- 处理器：双核 1.6 GHz 及以上
- 内存：4 GB 及以上
- 硬盘空间：200 MB 及以上可用空间
- 显示器：1280 x 720 及以上分辨率

**软件依赖：**
- 系统 WebView2 运行时（Windows 平台自动安装）
- 无需安装额外运行环境

## 2. 软件安装与卸载

### 2.1 安装说明

**Windows 平台：**

1. 获取安装包文件（.msi 或 .exe 格式）。
2. 双击安装包，启动安装向导。
3. 阅读并同意软件许可协议。
4. 选择安装目录（默认为 C:\Program Files\enote）。
5. 点击"安装"按钮，等待安装完成。
6. 安装完成后，可在桌面或开始菜单找到 enote 图标，双击启动。

**macOS 平台：**

1. 获取安装包文件（.dmg 格式）。
2. 双击 .dmg 文件，挂载磁盘映像。
3. 将 enote 应用图标拖拽至"应用程序"文件夹。
4. 在"应用程序"中找到 enote，双击启动。
5. 首次启动时，若系统提示安全确认，请前往"系统偏好设置 > 安全性与隐私"允许运行。

**Linux 平台：**

1. 获取安装包文件（.deb 或 .AppImage 格式）。
2. Debian/Ubuntu 系统：使用 `sudo dpkg -i enote_x.x.x_amd64.deb` 安装。
3. 其他发行版：赋予 AppImage 执行权限 `chmod +x enote_x.x.x.AppImage`，双击运行。

### 2.2 卸载说明

**Windows 平台：** 通过"控制面板 > 程序和功能"找到 enote，点击"卸载"。

**macOS 平台：** 将"应用程序"中的 enote 拖拽至废纸篓。

**Linux 平台：** Debian/Ubuntu 使用 `sudo apt remove enote` 卸载。

### 3.2 界面布局

enote 采用经典的三栏式布局设计，从左至右依次为：

**侧边栏（左侧）：** 宽度 256 像素，可折叠为 48 像素的图标栏。包含新建笔记按钮、笔记本列表、标签列表、导入按钮和语言切换按钮。

**笔记列表区（中间）：** 默认宽度 320 像素，支持拖拽调整宽度（200-600 像素范围），可完全折叠。包含当前笔记本名称标题、搜索栏、笔记卡片列表和分页控件。

**编辑器区域（右侧）：** 占据剩余全部宽度。顶部为功能丰富的编辑工具栏，下方为笔记标题输入框和正文编辑区，底部为状态栏显示光标位置与字数统计。

## 4. 笔记本管理

笔记本是 enote 中组织笔记的基本分类单元。每条笔记归属于一个笔记本，用户可通过笔记本对笔记进行分类管理。

### 4.1 查看笔记本列表

在侧边栏上方可以看到"NOTEBOOKS"标题下的笔记本列表。列表第一项固定为"全部"，表示查看所有笔记本中的笔记。每个笔记本项显示以下信息：

- **图标：** 用户选择的 Lucide 图标，若未设置则显示颜色标记圆点。
- **名称：** 笔记本的名称文字。
- **笔记数量：** 右侧显示该笔记本中包含的笔记数量。

### 4.2 新建笔记本

**操作步骤：**

1. 在侧边栏"NOTEBOOKS"标题右侧，点击菜单图标（三横线图标）。
2. 在弹出的下拉菜单中选择"新建"选项。
3. 系统弹出"新建笔记本"对话框，填写以下信息：
   - **名称（必填）：** 输入笔记本的名称。
   - **描述：** 输入笔记本的描述信息（可选）。
   - **图标：** 点击图标选择器，在搜索框中搜索或按分类浏览图标，选择一个合适的图标（可选）。图标选择器支持按名称搜索，图标按类别分组展示。
   - **排序：** 输入排序数值，数值越小排列越靠前。
4. 点击"确定"按钮完成创建。

### 4.3 编辑笔记本

**操作步骤：**

1. 在侧边栏笔记本列表中，点击选中要编辑的笔记本。
2. 点击"NOTEBOOKS"标题右侧的菜单图标。
3. 在下拉菜单中选择"编辑"选项（仅在选中非"全部"笔记本时可见）。
4. 在弹出的"编辑笔记本"对话框中修改相关信息。
5. 点击"确定"保存修改。

### 4.4 删除笔记本

**操作步骤：**

1. 选中要删除的笔记本。
2. 点击菜单图标，在下拉菜单中选择"删除"选项。
3. 系统弹出删除确认对话框，提示该操作将删除笔记本及其所有笔记。
4. 点击"确认"按钮执行删除，或点击"取消"放弃操作。

> **注意：** 删除笔记本将同时删除该笔记本中的所有笔记，请谨慎操作。"全部"虚拟笔记本不可删除。

### 4.5 按笔记本筛选

在侧边栏笔记本列表中，点击任一笔记本名称，笔记列表区域将自动筛选并显示该笔记本下的所有笔记。选中的笔记本项会高亮显示。点击"全部"可查看所有笔记本中的笔记。

## 5. 标签管理

标签是对笔记进行灵活标记和分类的辅助手段。一条笔记可以关联多个标签，便于从多个维度检索和管理笔记。

### 5.1 查看标签列表

在侧边栏笔记本列表下方，可以看到"TAGS"标题下的标签列表。列表第一项固定为"全部"，表示不按标签筛选。每个标签项显示图标或颜色标记以及标签名称。

### 5.2 新建标签

**操作步骤：**

1. 在侧边栏"TAGS"标题右侧，点击菜单图标。
2. 在下拉菜单中选择"新建"选项。
3. 系统弹出"新建标签"对话框，填写以下信息：
   - **名称（必填）：** 输入标签名称。
   - **图标：** 通过图标选择器选择一个图标（可选）。
   - **样式：** 通过样式选择器选择标签的颜色标记（可选），提供多种预设颜色供选择。
   - **排序：** 输入排序数值。
4. 点击"确定"完成创建。

### 5.3 编辑标签

**操作步骤：**

1. 在标签列表中选中要编辑的标签。
2. 点击"TAGS"标题右侧的菜单图标。
3. 选择"编辑"选项。
4. 在弹出的对话框中修改标签信息。
5. 点击"确定"保存。

### 5.4 删除标签

**操作步骤：**

1. 选中要删除的标签。
2. 点击菜单图标，选择"删除"选项。
3. 在确认对话框中点击"确认"执行删除。

> **注意：** 删除标签不会删除关联的笔记，仅移除标签与笔记之间的关联关系。

### 5.5 按标签筛选

点击侧边栏中的任一标签，笔记列表将筛选显示关联了该标签的所有笔记。

## 6. 笔记管理

### 6.1 笔记列表

笔记列表位于界面中间区域，展示当前筛选条件下的所有笔记。每条笔记卡片显示以下信息：

- **标题：** 加粗显示，单行截断。若未设置标题则显示"无标题"。
- **内容摘要：** 显示最多两行的纯文本预览，自动去除 HTML 和 Markdown 标记。
- **所属笔记本：** 左下角显示笔记归属的笔记本名称。
- **更新时间：** 右下角显示最后修改时间。

当前选中的笔记卡片会高亮显示。支持键盘方向键上下移动选择，按回车键或空格键确认选中。

**分页：** 列表底部提供分页控件，支持翻页浏览大量笔记。

**空状态：** 当没有笔记时显示引导提示"还没有笔记，点击创建新笔记"；搜索无结果时显示"没有找到匹配的笔记"。

**列表宽度调整：** 拖动笔记列表区域右侧边缘可调整列表宽度，支持 200 至 600 像素范围。

**列表面板折叠/展开：** 笔记列表右侧边缘中央有一个圆形折叠按钮，点击可完全折叠笔记列表面板，为编辑器腾出更多空间。再次点击可展开列表，展开后宽度恢复为默认的 320 像素。

### 6.2 新建笔记

**方式一：** 点击侧边栏顶部的"添加笔记"按钮（侧边栏展开时显示为完整按钮，折叠时显示为圆形加号图标）。

**方式二：** 使用键盘快捷键 Ctrl+N（macOS 为 Cmd+N）。

新建笔记时：
1. 笔记自动归属于当前选中的笔记本。
2. 编辑器进入编辑模式，光标自动定位到标题输入框。
3. 用户需在工具栏左侧选择内容类型：
   - **富文本：** 使用所见即所得的富文本编辑器。
   - **Markdown：** 使用 Markdown 语法编辑器。

> **注意：** 笔记的内容类型在首次保存后不可更改。

### 6.3 编辑笔记

**操作步骤：**

1. 在笔记列表中点击选择要编辑的笔记，编辑器区域显示笔记内容（只读模式）。
2. 点击工具栏右侧的"编辑"按钮（铅笔图标），或使用快捷键 Ctrl+E（macOS 为 Cmd+E）进入编辑模式。
3. 编辑笔记标题和内容。
4. 编辑完成后点击"保存"按钮或使用快捷键 Ctrl+S（macOS 为 Cmd+S）保存。
5. 如需放弃更改，点击"取消"按钮或按 Esc 键。

> **注意：** 对于新建但尚未保存的笔记，点击"取消"后笔记将从列表中自动移除，不会保留空白笔记。对于已保存的笔记，取消编辑后内容恢复到上次保存的状态。

### 6.4 保存笔记

在编辑模式下，点击工具栏右侧的"保存"按钮（对勾图标）或使用快捷键 Ctrl+S（macOS 为 Cmd+S）保存当前笔记。保存成功后系统显示成功通知，笔记自动切换为只读模式。

系统会自动记录每次保存的历史版本，便于后续版本追溯和内容恢复。

### 6.5 删除笔记

**操作步骤：**

1. 选中要删除的笔记。
2. 点击工具栏右侧的"删除"按钮（红色垃圾桶图标）。
3. 在确认对话框中点击"确认"执行删除。

删除操作会自动创建历史记录，记录被删除的笔记内容。

### 6.6 搜索笔记

笔记列表区域顶部提供搜索栏：

1. 在搜索框中输入关键词。
2. 按下回车键执行搜索。
3. 笔记列表将筛选显示标题或内容中包含关键词的笔记。
4. 搜索框右侧出现清除按钮（X 图标），点击可清除搜索条件并恢复完整列表。

搜索支持同时匹配笔记标题和笔记内容。

### 6.7 笔记设置

在编辑模式下，点击工具栏的"设置"按钮（齿轮图标）可打开笔记设置对话框，用于管理笔记的归属和标签：

- **所属笔记本：** 通过可搜索的下拉选择器，将笔记移动到指定笔记本。
- **标签：** 以网格形式展示所有可用标签，以勾选框形式选中或取消关联的标签。一条笔记可关联多个标签。

修改完成后点击"保存"按钮生效。

## 7. 富文本编辑器

enote 集成了功能丰富的所见即所得（WYSIWYG）富文本编辑器，工具栏位于编辑器顶部，当工具栏按钮过多时支持水平滚动，左右两侧有滚动指示箭头。

### 7.1 文本样式

工具栏提供以下文本格式化按钮：

| 按钮 | 功能 | 说明 |
|------|------|------|
| **B** | 粗体 | 将选中文本设为粗体 |
| *I* | 斜体 | 将选中文本设为斜体 |
| U | 下划线 | 为选中文本添加下划线 |
| ~~S~~ | 删除线 | 为选中文本添加删除线 |
| X² | 上标 | 将选中文本设为上标 |
| X₂ | 下标 | 将选中文本设为下标 |

点击按钮可切换对应格式的开启/关闭状态，选中文本后点击按钮应用格式，已应用的格式按钮会高亮显示。

### 7.2 标题与字体

**标题层级：** 通过下拉选择器设置段落的标题级别，支持：
- 正文（默认段落）
- 标题 1（H1）至 标题 6（H6），字号从大到小递减。

**字体族：** 下拉选择器提供多种字体选择，按类别分组：
- **无衬线体：** Arial、Helvetica、Verdana、Tahoma、Trebuchet MS、微软雅黑、苹方
- **衬线体：** Times New Roman、Georgia、Palatino、宋体、楷体、仿宋
- **等宽字体：** Courier New、Consolas、Monaco、Source Code Pro
- **艺术字体：** Comic Sans MS、Impact、Brush Script

**字体大小：** 下拉选择器支持 12px、14px、16px、18px、20px、24px、28px、32px、36px、48px、72px 等多种预设字号。

### 7.3 文本对齐

提供四种段落对齐方式按钮：

- **左对齐：** 文本靠左排列（默认）。
- **居中对齐：** 文本居中排列。
- **右对齐：** 文本靠右排列。
- **两端对齐：** 文本两端对齐排列。

### 7.4 列表功能

支持三种列表类型：

- **无序列表：** 以项目符号标记的列表。
- **有序列表：** 以数字编号标记的列表。
- **任务列表：** 带有复选框的任务清单列表，点击复选框可切换完成状态。已完成的任务项显示删除线和灰色文字效果。支持任务列表嵌套。

### 7.5 缩进控制

- **增加缩进：** 点击增加缩进按钮或按 Tab 键，段落向右缩进一级（每级 2em）。
- **减少缩进：** 点击减少缩进按钮或按 Shift+Tab 键，段落向左减少一级缩进。
- 最大缩进层级为 10 级，最小为 0 级。

### 7.6 引用与代码

- **引用块：** 点击引用按钮，将段落转为引用块样式，左侧显示主题色竖线，文本以灰色斜体显示。
- **代码块：** 点击代码块按钮，插入带语法高亮的代码块。支持多种编程语言的语法着色显示（基于 highlight.js 引擎）。
- **行内代码：** 点击行内代码按钮，将选中文本设为等宽字体的行内代码样式。

### 7.7 链接与图片

**插入链接：**
1. 选中需要添加链接的文本。
2. 点击工具栏的链接按钮。
3. 在弹出的对话框中输入链接地址（URL）。
4. 点击"确定"插入链接。链接文本以蓝色下划线样式显示。

**取消链接：** 选中已有链接的文本，点击"取消链接"按钮移除超链接。

**插入图片：**
1. 点击工具栏的图片按钮。
2. 在弹出的对话框中选择插入方式：
   - **输入 URL：** 在文本框中输入图片的网络地址。
   - **上传文件：** 点击上传按钮选择本地图片文件，支持常见图片格式。
3. 点击"确定"插入图片。

图片在编辑器中以圆角样式显示，支持懒加载技术，加载过程中显示动画占位符。

### 7.8 表格功能

- **插入表格：** 点击插入表格按钮，自动插入一个 3 行 3 列的表格。
- **添加列：** 在当前列后方插入新列。
- **添加行：** 在当前行下方插入新行。
- **删除表格：** 删除整个表格。
- 表格支持通过拖拽列边框调整列宽。选中的单元格会高亮显示。

### 7.9 颜色设置

**文字颜色：** 点击文字颜色按钮打开颜色选择器，提供 10 种预设颜色的宫格选择，同时支持通过颜色选择器或输入十六进制色值自定义颜色。

**背景颜色：** 点击背景颜色按钮，为选中文本设置背景高亮色，颜色选择方式同文字颜色。

**荧光笔高亮：** 点击高亮按钮，快速为选中文本添加黄色高亮背景。

### 7.10 查找与替换

点击工具栏的"查找替换"按钮，在编辑器上方打开查找替换面板：

- **查找输入框：** 输入要查找的文本，按回车键定位到下一个匹配项。
- **替换输入框：** 输入替换后的文本。
- **区分大小写：** 复选框控制是否区分大小写匹配。
- **结果计数：** 显示当前匹配位置和总匹配数（如"3 / 12"）。
- **上一个/下一个：** 箭头按钮在匹配结果间导航。
- **替换：** 替换当前匹配项。
- **全部替换：** 一次性替换所有匹配项。

所有匹配项在编辑器中以黄色高亮显示，当前聚焦的匹配项以橙色高亮区分。

### 7.11 目录导航

点击工具栏的"目录"按钮，在编辑器右侧打开浮动目录导航面板（宽度 240 像素）：

- 自动提取文档中的所有标题（H1-H6），按层级缩进展示。
- 点击目录中的任一条目，编辑器自动滚动定位到对应的标题位置。
- 当前可视区域内的标题在目录中高亮显示（左侧显示主题色竖线）。
- 点击面板右上角的关闭按钮（X）关闭目录面板。

### 7.12 撤销与重做

- **撤销：** 点击撤销按钮，回退到上一步编辑操作。
- **重做：** 点击重做按钮，恢复被撤销的操作。

### 7.13 清除格式

点击工具栏的"清除格式"按钮，移除选中文本的所有格式化样式，恢复为默认文本格式。

### 7.14 内容块拖拽排序

富文本编辑器支持通过拖拽方式调整内容块的顺序。将鼠标悬停在段落、标题、引用块、代码块等块级元素上时，元素左侧会出现拖拽手柄。按住拖拽手柄并拖动，可将该内容块移动到文档中的其他位置。拖动过程中，编辑器会显示当前拖拽状态的视觉指示，释放鼠标后内容块将插入到目标位置。

此功能适用于快速调整文档结构，无需通过剪切粘贴操作即可重新排列内容。

### 7.15 数学公式

富文本编辑器支持数学公式的渲染显示，基于 KaTeX 数学排版引擎。用户可以在笔记中插入数学公式，系统自动将公式语法渲染为标准的数学符号格式，支持行内公式和独立公式块，满足学术笔记和技术文档的数学表达需求。

### 7.16 图片懒加载

编辑器中的图片采用智能懒加载技术。当笔记包含较多图片时，仅加载当前可视区域内的图片，未进入可视区域的图片显示为带有动画效果的骨架占位符。当用户滚动页面使图片进入可视范围时，图片自动开始加载并以渐入动画显示。此机制有效减少初始加载时间，提升长文档的浏览流畅度。本地上传的图片（Base64 格式）不受懒加载限制，会立即显示。

## 8. Markdown 编辑器

当笔记内容类型选择为 Markdown 时，编辑器提供专业的 Markdown 编写体验。

### 8.1 源码编辑模式

在源码模式下，编辑区域显示为深色主题的等宽字体文本编辑区，用户直接编写 Markdown 语法文本。支持标准 Markdown 语法，包括标题、列表、链接、图片、代码块、表格、引用等。

### 8.2 预览模式

点击工具栏的"源码/预览切换"按钮，可在 Markdown 源码编辑和渲染预览之间切换。预览模式下显示 Markdown 语法渲染后的格式化内容。

Markdown 预览支持以下增强渲染：
- **数学公式：** 基于 KaTeX 引擎渲染数学公式。使用 `$...$` 编写行内公式，使用 `$$...$$` 编写独立公式块。
- **语法高亮：** 代码块自动识别编程语言并着色显示。
- **表格、任务列表：** 标准 Markdown 扩展语法均可正常渲染。

### 8.3 分屏模式

Markdown 编辑器支持两种分屏布局，方便同时查看源码和预览效果：

**垂直分屏：** 点击工具栏的"垂直布局"按钮，编辑区上方显示 Markdown 源码，下方显示渲染预览。

**水平分屏：** 点击工具栏的"水平布局"按钮，编辑区左侧显示 Markdown 源码，右侧显示渲染预览。

分屏模式特性：
- 中间分隔线支持拖拽调整两个面板的比例（范围 20%-80%）。
- 源码区和预览区支持同步滚动，编辑时预览自动跟随定位。
- 大文档使用 Web Worker 技术进行异步预处理，确保编辑流畅。

## 9. 笔记历史记录

enote 自动记录笔记的每次修改和删除操作历史，支持版本追溯和内容对比。

### 9.1 查看历史列表

**操作步骤：**

1. 选中一条笔记。
2. 点击工具栏右侧的"历史记录"按钮（时钟图标）。
3. 系统弹出全屏历史记录对话框，以表格形式展示历史记录列表。

历史记录表格包含以下列信息：

| 列名 | 说明 |
|------|------|
| ID | 历史记录的唯一标识 |
| 笔记本 | 笔记所属笔记本名称 |
| 标题 | 笔记标题 |
| 内容类型 | 富文本或 Markdown |
| 标签 | 关联的标签 |
| 操作类型 | 新增、修改或删除 |
| 操作时间 | 操作发生的时间 |
| 操作 | 查看详情按钮 |

列表支持分页浏览，每页可选择显示 20、50、100 或 200 条记录。加载过程中显示骨架屏占位动画。

### 9.2 查看历史内容对比

在历史记录列表中，点击某条记录的"查看"按钮，打开内容对比查看窗口：

- **左侧面板（灰色标题）：** 显示修改前的旧内容。
- **右侧面板（主题色标题）：** 显示修改后的新内容。

两侧内容均以富文本格式渲染显示，方便直观对比修改差异。

## 10. 导出功能

enote 支持将笔记导出为多种格式，满足不同的使用场景和数据迁移需求。

**操作步骤：**

1. 选中要导出的笔记。
2. 点击工具栏右侧的"导出"按钮（下载图标）。
3. 在弹出的导出对话框中选择目标格式。
4. 点击"导出"按钮。
5. 系统弹出文件保存对话框，选择保存位置和文件名。
6. 导出完成后显示成功通知。

### 10.1 导出为 Word 文档

将笔记导出为 .doc 格式的 Word 文档，保留文本格式和样式。文档顶部包含笔记标题作为主标题，支持在 Microsoft Word、WPS 等办公软件中打开编辑。

### 10.2 导出为 Markdown

将笔记导出为 .md 格式的 Markdown 文件。富文本类型的笔记会自动转换为 Markdown 语法；Markdown 类型的笔记直接导出原始内容。

### 10.3 导出为印象笔记格式

将笔记导出为 .enex 格式，兼容印象笔记（Evernote）的导入功能，便于将笔记迁移至印象笔记平台。

### 10.4 导出为 JSON

将笔记导出为 .json 格式的结构化数据文件，包含以下字段：
- 标题（title）
- 内容（content）
- 内容类型（contentType）
- 标签（tags）
- 创建时间（createTime）
- 更新时间（updateTime）

### 10.5 导出为 XML

将笔记导出为 .xml 格式，数据结构与 JSON 导出一致，适用于需要 XML 格式数据交换的场景。

## 11. 导入功能

enote 支持从多个主流笔记平台导入数据，帮助用户便捷地迁移已有笔记。

点击侧边栏底部的"导入"按钮启动导入向导，共分四个步骤。

### 11.1 从印象笔记导入

**操作步骤：**

1. **选择来源：** 在导入向导第一步选择"印象笔记"。
2. **选择文件：** 将从印象笔记导出的 .enex 文件拖拽至上传区域，或点击区域选择文件。
3. **选择目标笔记本：** 在下拉选择器中选择要导入到的目标笔记本。可选择勾选"自动创建不存在的标签"选项（默认勾选）。
4. **执行导入：** 点击"开始导入"按钮。系统显示导入进度，包括当前处理的笔记标题和进度百分比。
5. **导入完成：** 显示导入结果摘要，包括成功数量和失败详情。

### 11.2 从有道笔记导入

**操作步骤：**

1. **选择来源：** 选择"有道笔记"。
2. **选择文件：** 选择从有道笔记导出的 .zip 压缩包。
3. **选择目标笔记本并配置选项。**
4. **执行导入并查看结果。**

### 11.3 从 Notion 导入

**操作步骤：**

1. **选择来源：** 选择"Notion"。
2. **选择文件：** 选择从 Notion 导出的 .zip 压缩包（包含 Markdown 文件）。
3. **选择目标笔记本并配置选项。**
4. **执行导入并查看结果。**

> **注意：** 导入过程中会自动解析源文件中的标签信息。若勾选"自动创建不存在的标签"，系统将自动创建源数据中存在但系统中不存在的标签。

## 15. 应用启动与初始化

### 15.1 首次启动

首次启动 enote 时，系统自动完成以下初始化操作：

1. **创建配置文件：** 在应用数据目录下自动生成 `application.yml` 配置文件，包含默认的数据库连接设置。
2. **创建数据库：** 自动创建 SQLite 数据库文件，并执行数据表结构初始化（自动迁移）。
3. **加载数据：** 并行加载笔记本列表、标签列表和笔记数据。

启动过程中，界面显示加载提示通知，加载完成后通知自动消失。

### 15.2 启动异常处理

若启动过程中遇到异常，系统会弹出操作系统原生错误对话框，提示具体的错误信息：

- **配置文件加载失败：** 提示配置文件读取错误，请检查 `application.yml` 格式是否正确。
- **数据库连接失败：** 提示数据库连接异常，请检查数据库配置信息或数据库服务是否可用。
- **数据库迁移失败：** 提示数据表结构更新异常。

错误提示信息支持中英文国际化，根据系统语言环境自动显示对应语言。确认错误对话框后应用将自动退出，用户需排查并修复问题后重新启动。

### 15.3 后续启动

非首次启动时，系统自动检测并执行数据库结构升级（如有新版本的表结构变更），确保数据兼容性。整个过程对用户透明，无需手动干预。

## 16. 数据存储与配置

### 16.1 本地数据存储

所有笔记数据默认存储在本地 SQLite 数据库中，数据完全保存在用户本地设备上，不经过任何网络传输，保障数据隐私安全。

应用配置文件存储在操作系统的应用数据目录中。使用默认 SQLite 数据库时，数据库文件也位于该目录下；使用 MySQL 或 PostgreSQL 时，数据存储在对应的数据库服务器上，该目录下仅保存配置文件。不同操作系统的应用数据目录路径如下：

| 操作系统 | 应用数据目录 | 示例 |
|----------|-------------|------|
| **Windows** | `{FOLDERID_RoamingAppData}\net.easycloudcn.enote\` | `C:\Users\Alice\AppData\Roaming\net.easycloudcn.enote\` |
| **macOS** | `$HOME/Library/Application Support/net.easycloudcn.enote/` | `/Users/Alice/Library/Application Support/net.easycloudcn.enote/` |
| **Linux** | `$XDG_DATA_HOME/net.easycloudcn.enote/` 或 `$HOME/.local/share/net.easycloudcn.enote/` | `/home/alice/.local/share/net.easycloudcn.enote/` |

> **快速访问：**
> - **Windows：** 在文件资源管理器地址栏输入 `%APPDATA%\net.easycloudcn.enote` 即可快速打开。
> - **macOS：** 在 Finder 中按 `Cmd+Shift+G`，输入 `~/Library/Application Support/net.easycloudcn.enote` 即可打开。
> - **Linux：** 在终端中执行 `xdg-open "${XDG_DATA_HOME:-$HOME/.local/share}/net.easycloudcn.enote"` 即可打开。

该目录下包含以下文件：

| 文件 | 说明 |
|------|------|
| `application.yml` | 应用配置文件（首次启动自动生成），所有数据库类型均存在 |
| `enote.db` | SQLite 数据库文件，仅使用默认 SQLite 配置时存在 |
| `enote.db-wal` | SQLite WAL 日志文件（自动生成，用于提升并发性能），仅 SQLite 时存在 |
| `enote.db-shm` | SQLite 共享内存文件（自动生成），仅 SQLite 时存在 |

### 16.2 配置文件

应用配置文件为 `application.yml`，采用 YAML 格式，位于上述应用数据目录中。首次启动时系统会自动生成包含 SQLite 默认配置的文件，用户无需手动创建。

**默认生成的配置文件内容如下：**

```yaml
# ENote 应用配置文件
# 此文件由应用自动生成

datasource:
  # SQLite 数据库连接 URL
  # mode=rwc 表示读写模式，如果文件不存在则创建
  url: "sqlite:/path/to/net.easycloudcn.enote/enote.db?mode=rwc"

  # 连接池配置
  # 最大连接数：SQLite 推荐 5 以内
  max-connections: 5
  # 最小连接数：保持常驻连接减少重连开销
  min-connections: 1
  # 连接超时（秒）：建立新连接的超时时间
  connect_timeout: 10
  # 获取连接超时（秒）：从池中获取连接的超时时间
  acquire-timeout: 5
  # 空闲超时（秒）：空闲连接保持时间
  idle-time: 300
  # 最大生命周期（秒）：连接最长存活时间
  max-lifetime: 1800
```

> **注意：** 配置文件中 `url` 字段的实际路径会根据操作系统自动填入完整的绝对路径。修改配置文件后需**重启应用**使配置生效。

### 16.3 配置项说明

以下为 `datasource` 节点下所有可配置项的详细说明：

| 配置项 | 类型 | 默认值 | 有效范围 | 说明 |
|--------|------|--------|----------|------|
| `url` | 字符串 | （自动生成） | — | **必填**。数据库连接 URL，格式取决于数据库类型（见 16.4 节） |
| `max-connections` | 整数 | SQLite: 5，其他: 20 | 1 - 1000 | 连接池最大连接数。SQLite 建议不超过 5 |
| `min-connections` | 整数 | 1 | 0 - max-connections | 连接池最小保持连接数，设为 0 表示不保持常驻连接 |
| `connect_timeout` | 整数 | 10 | 1 - 300 | 建立新数据库连接的超时时间（秒） |
| `acquire-timeout` | 整数 | 5 | 1 - 300 | 从连接池获取可用连接的超时时间（秒） |
| `idle-time` | 整数 | 300 | 1 - 86400 | 空闲连接保持时间（秒），超过此时间未使用的连接将被回收 |
| `max-lifetime` | 整数 | 1800 | 1 - 86400 | 连接最大存活时间（秒），超过此时间的连接将被强制关闭并重建 |

### 16.4 数据库配置示例

enote 支持三种数据库，通过修改 `datasource.url` 即可切换。除默认的 SQLite 外，还支持 MySQL 和 PostgreSQL，满足高级用户的数据管理需求。

**SQLite（默认）：**

```yaml
datasource:
  url: "sqlite:/path/to/enote.db?mode=rwc"
  max-connections: 5
  min-connections: 1
```

- `mode=rwc`：读写模式，数据库文件不存在时自动创建。
- SQLite 为单文件数据库，无需安装额外服务，适合个人使用。

**MySQL：**

```yaml
datasource:
  url: "mysql://用户名:密码@服务器地址:3306/数据库名"
  max-connections: 20
  min-connections: 2
  connect_timeout: 10
  acquire-timeout: 5
  idle-time: 300
  max-lifetime: 1800
```

- 需预先安装 MySQL 5.7+ 或 MariaDB 10.3+ 并创建目标数据库。
- 示例：`url: "mysql://root:password@127.0.0.1:3306/enote"`

**PostgreSQL：**

```yaml
datasource:
  url: "postgres://用户名:密码@服务器地址:5432/数据库名"
  max-connections: 20
  min-connections: 2
  connect_timeout: 10
  acquire-timeout: 5
  idle-time: 300
  max-lifetime: 1800
```

- 需预先安装 PostgreSQL 12+ 并创建目标数据库。
- 示例：`url: "postgres://enote:password@127.0.0.1:5432/enote"`

> **注意：** 切换数据库类型后，原有数据库中的数据不会自动迁移。如需保留数据，请先使用导出功能备份笔记，切换后再导入。首次连接新数据库时，系统会自动创建所需的表结构。

### 16.5 历史版本保护

系统自动记录笔记的每一次修改和删除操作的历史版本，包括操作前后的完整内容，支持随时追溯和对比，有效防止数据意外丢失。历史记录包含以下信息：

- 操作类型（新增、修改、删除）
- 操作前的旧内容
- 操作后的新内容
- 操作发生时间
- 关联的笔记本和标签信息
