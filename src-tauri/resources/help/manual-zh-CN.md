## 1. 引言

### 1.1 编写目的

本手册旨在详细介绍 ENote 智能笔记管理系统的各项功能和操作方法，帮助用户快速了解和掌握本软件的使用方式，充分利用软件提供的笔记管理能力，提高日常信息记录和知识管理的效率。

### 1.2 软件概述

ENote 智能笔记管理系统是一款跨平台桌面笔记应用程序，为用户提供高效、便捷的笔记创建、编辑、管理和检索功能。软件支持富文本和 Markdown 两种编辑模式，具备笔记本分类管理、标签体系、全文搜索、历史版本追溯、多格式导入导出、笔记加密、双向链接、模板系统、自动备份等核心功能，满足个人用户对知识管理和信息记录的全面需求。

软件采用前后端分离架构，前端基于 Vue 3 框架和 TypeScript 语言开发，后端采用 Rust 语言和 Tauri 2.x 框架，数据存储层使用 SeaORM 对象关系映射框架，默认支持 SQLite 数据库，同时兼容 MySQL 和 PostgreSQL 数据库。整体架构保证了软件的高性能、低资源占用和跨平台兼容性。

### 1.3 运行环境

**操作系统：**
- Windows 10 及以上版本（64位）
- macOS 10.15 (Catalina) 及以上版本
- Linux（主流发行版，如 Ubuntu 20.04+、Fedora 34+）

**硬件要求：**
- 处理器：双核 1.6 GHz 及以上
- 内存：4 GB 及以上
- 硬盘空间：200 MB 及以上可用空间
- 显示器：1280 x 720 及以上分辨率

**软件依赖：**
- 系统 WebView2 运行时（Windows 平台自动安装）
- 无需安装额外运行环境

## 2. 软件安装与卸载

### 2.1 安装说明

**Windows 平台：**

1. 获取安装包文件（.msi 或 .exe 格式）。
2. 双击安装包，启动安装向导。
3. 阅读并同意软件许可协议。
4. 选择安装目录（默认为 C:\Program Files\enote）。
5. 点击"安装"按钮，等待安装完成。
6. 安装完成后，可在桌面或开始菜单找到 ENote 图标，双击启动。

**macOS 平台：**

1. 获取安装包文件（.dmg 格式）。
2. 双击 .dmg 文件，挂载磁盘映像。
3. 将 ENote 应用图标拖拽至"应用程序"文件夹。
4. 在"应用程序"中找到 ENote，双击启动。
5. 首次启动时，若系统提示安全确认，请前往"系统偏好设置 > 安全性与隐私"允许运行。

**Linux 平台：**

1. 获取安装包文件（.deb 或 .AppImage 格式）。
2. Debian/Ubuntu 系统：使用 `sudo dpkg -i enote_x.x.x_amd64.deb` 安装。
3. 其他发行版：赋予 AppImage 执行权限 `chmod +x enote_x.x.x.AppImage`，双击运行。

### 2.2 卸载说明

**Windows 平台：** 通过"控制面板 > 程序和功能"找到 ENote，点击"卸载"。

**macOS 平台：** 将"应用程序"中的 ENote 拖拽至废纸篓。

**Linux 平台：** Debian/Ubuntu 使用 `sudo apt remove enote` 卸载。

### 3.2 界面布局

ENote 采用响应式三栏布局设计，根据屏幕宽度自动适配不同设备：

#### 桌面模式（宽度 > 1024px）

三栏并排显示，从左至右依次为：

**侧边栏（左侧）：** 宽度 256 像素，可折叠为 48 像素的图标栏。包含新建笔记按钮、笔记本列表、标签列表，底部工具栏包含导入笔记、数据备份、模板管理、设置和回收站等图标按钮（鼠标悬浮显示功能提示）。

**笔记列表区（中间）：** 默认宽度 320 像素，支持拖拽调整宽度（200-600 像素范围），可完全折叠。包含当前笔记本名称标题、搜索栏、笔记卡片列表和分页控件。置顶的笔记在列表顶部显示，带有置顶标识。

**编辑器区域（右侧）：** 占据剩余全部宽度。顶部为功能丰富的编辑工具栏，下方为笔记标题输入框和正文编辑区，编辑区下方为关联笔记面板（可展开/折叠），底部为状态栏显示光标位置与字数统计。

#### 平板模式（宽度 640px ~ 1024px）

双栏显示：笔记列表（固定 320px）+ 编辑器。侧边栏以覆盖层形式弹出，点击笔记列表左上角的菜单图标打开，点击遮罩层或关闭按钮收起。拖拽调整宽度和折叠按钮在此模式下隐藏。

#### 手机模式（宽度 < 640px）

单视图全屏切换：笔记列表和编辑器分别全屏显示，通过滑动动画切换。在笔记列表中选择笔记后自动切换到编辑器视图，编辑器左上角显示返回按钮回到列表。侧边栏同样以覆盖层形式弹出。

> **提示：** 布局模式可在设置中手动切换（自动/桌面/平板/手机），也可通过快捷键 Ctrl+Shift+M 循环切换。详见 [14.3 布局模式设置](#143-布局模式设置)。

## 4. 笔记本管理

笔记本是 ENote 中组织笔记的基本分类单元。每条笔记归属于一个笔记本，用户可通过笔记本对笔记进行分类管理。

### 4.1 查看笔记本列表

在侧边栏上方可以看到"NOTEBOOKS"标题下的笔记本列表。列表第一项固定为"全部"，表示查看所有笔记本中的笔记。每个笔记本项显示以下信息：

- **图标：** 用户选择的 Lucide 图标，若未设置则显示颜色标记圆点。
- **名称：** 笔记本的名称文字。
- **笔记数量：** 右侧显示该笔记本中包含的笔记数量。

### 4.2 新建笔记本

**操作步骤：**

1. 在侧边栏"NOTEBOOKS"标题右侧，点击菜单图标（三横线图标）。
2. 在弹出的下拉菜单中选择"新建"选项。
3. 系统弹出"新建笔记本"对话框，填写以下信息：
   - **名称（必填）：** 输入笔记本的名称。
   - **描述：** 输入笔记本的描述信息（可选）。
   - **图标：** 点击图标选择器，在搜索框中搜索或按分类浏览图标，选择一个合适的图标（可选）。图标选择器支持按名称搜索，图标按类别分组展示。
   - **排序：** 输入排序数值，数值越小排列越靠前。
   - **AI 访问权限：** 设置 AI 工具通过 MCP 协议对该笔记本下笔记的访问级别（可选）。可选值包括"继承"（默认）、"读写"、"只读"、"禁止"，详见 [20.4 权限控制](#204-权限控制)。
4. 点击"确定"按钮完成创建。

> **提示：** 设置了非"继承"AI 访问权限的笔记本，在侧边栏列表中会显示一个盾牌图标（🛡），便于快速识别。

### 4.3 编辑笔记本

**操作步骤：**

1. 在侧边栏笔记本列表中，点击选中要编辑的笔记本。
2. 点击"NOTEBOOKS"标题右侧的菜单图标。
3. 在下拉菜单中选择"编辑"选项（仅在选中非"全部"笔记本时可见）。
4. 在弹出的"编辑笔记本"对话框中修改相关信息。
5. 点击"确定"保存修改。

### 4.4 删除笔记本

**操作步骤：**

1. 选中要删除的笔记本。
2. 点击菜单图标，在下拉菜单中选择"删除"选项。
3. 系统弹出删除确认对话框，提示该操作将删除笔记本及其所有笔记。
4. 点击"确认"按钮执行删除，或点击"取消"放弃操作。

> **注意：** 删除笔记本将同时删除该笔记本中的所有笔记，请谨慎操作。"全部"虚拟笔记本不可删除。

### 4.5 按笔记本筛选

在侧边栏笔记本列表中，点击任一笔记本名称，笔记列表区域将自动筛选并显示该笔记本下的所有笔记。选中的笔记本项会高亮显示。点击"全部"可查看所有笔记本中的笔记。

### 4.6 拖拽排序笔记本

侧边栏笔记本列表支持拖拽排序。按住笔记本项并上下拖动即可调整排列顺序，释放鼠标后新的排序自动保存到数据库。"全部"虚拟笔记本始终固定在首位，不参与排序。

   - **AI 访问权限：** 设置 AI 工具对带有该标签的笔记的访问级别（可选）。可选值包括"继承"（默认）、"读写"、"只读"、"禁止"，详见 [20.4 权限控制](#204-权限控制)。

> **提示：** 设置了非"继承"AI 访问权限的标签，在侧边栏列表中会显示一个盾牌图标（🛡），便于快速识别。

### 5.3 编辑标签

### 5.4 删除标签

### 5.5 按标签筛选

点击侧边栏中的任一标签，笔记列表将筛选显示关联了该标签的所有笔记。

### 5.6 拖拽排序标签

与笔记本类似，标签列表也支持拖拽排序。按住标签项上下拖动即可调整顺序，释放后自动保存。

## 6. 笔记管理

### 6.1 笔记列表

笔记列表位于界面中间区域，展示当前筛选条件下的所有笔记。每条笔记卡片显示以下信息：

- **置顶标识：** 已置顶的笔记在卡片上显示置顶图标，并始终排列在列表顶部。
- **标题：** 加粗显示，单行截断。若未设置标题则显示"无标题"。
- **内容摘要：** 显示最多两行的纯文本预览，自动去除 HTML 和 Markdown 标记。
- **所属笔记本：** 左下角显示笔记归属的笔记本名称。
- **更新时间：** 右下角显示最后修改时间。

当前选中的笔记卡片会高亮显示。支持键盘方向键上下移动选择，按回车键或空格键确认选中。

**分页：** 列表底部提供分页控件，支持翻页浏览大量笔记。

**空状态：** 当没有笔记时显示引导提示"还没有笔记，点击创建新笔记"；搜索无结果时显示"没有找到匹配的笔记"。

**列表宽度调整：** 拖动笔记列表区域右侧边缘可调整列表宽度，支持 200 至 600 像素范围。

**列表面板折叠/展开：** 笔记列表右侧边缘中央有一个圆形折叠按钮，点击可完全折叠笔记列表面板，为编辑器腾出更多空间。再次点击可展开列表，展开后宽度恢复为默认的 320 像素。

### 6.2 新建笔记

**方式一：** 点击侧边栏顶部的"添加笔记"按钮（侧边栏展开时显示为完整按钮，折叠时显示为圆形加号图标）。

**方式二：** 使用键盘快捷键 Ctrl+N（macOS 为 Cmd+N）。

**方式三：** 使用命令面板（Ctrl+P），搜索并选择"新建笔记"命令。

新建笔记时：
1. 笔记自动归属于当前选中的笔记本。
2. 编辑器进入编辑模式，光标自动定位到标题输入框。
3. 用户需在工具栏左侧选择内容类型：
   - **富文本：** 使用所见即所得的富文本编辑器。
   - **Markdown：** 使用 Markdown 语法编辑器。

> **注意：** 笔记的内容类型在首次保存后不可更改。

### 6.3 编辑笔记

**操作步骤：**

1. 在笔记列表中点击选择要编辑的笔记，编辑器区域显示笔记内容（只读模式）。
2. 点击工具栏右侧的"编辑"按钮（铅笔图标），或使用快捷键 Ctrl+E（macOS 为 Cmd+E）进入编辑模式。
3. 编辑笔记标题和内容。
4. 编辑完成后点击"保存"按钮或使用快捷键 Ctrl+S（macOS 为 Cmd+S）保存。
5. 如需放弃更改，点击"取消"按钮或按 Esc 键。

> **注意：** 对于新建但尚未保存的笔记，点击"取消"后笔记将从列表中自动移除，不会保留空白笔记。对于已保存的笔记，取消编辑后内容恢复到上次保存的状态。

### 6.4 保存笔记

在编辑模式下，点击工具栏右侧的"保存"按钮（对勾图标）或使用快捷键 Ctrl+S（macOS 为 Cmd+S）保存当前笔记。保存成功后系统显示成功通知，笔记自动切换为只读模式。

系统会自动记录每次保存的历史版本，便于后续版本追溯和内容恢复。

### 6.5 删除笔记与回收站

**删除笔记（移入回收站）：**

1. 选中要删除的笔记。
2. 点击工具栏右侧的"删除"按钮（红色垃圾桶图标）。
3. 在确认对话框中点击"确认"执行删除。

删除后的笔记不会立即从数据库移除，而是被移入回收站（软删除）。

**回收站操作：**

点击侧边栏底部的"回收站"图标按钮打开回收站对话框，可以进行以下操作：

- **查看已删除笔记：** 回收站列表显示所有被删除的笔记，包含标题和删除时间。
- **恢复笔记：** 点击笔记右侧的"恢复"按钮，将笔记恢复到原来的笔记本中。
- **永久删除：** 点击"永久删除"按钮，彻底删除单条笔记，此操作不可恢复。
- **清空回收站：** 点击"清空回收站"按钮，永久删除回收站中的所有笔记。系统会弹出确认对话框，确认后执行清空操作，此操作不可恢复。

### 6.6 搜索笔记

笔记列表区域顶部提供搜索栏：

1. 在搜索框中输入关键词。
2. 按下回车键执行搜索。
3. 笔记列表将筛选显示标题或内容中包含关键词的笔记。
4. 搜索框右侧出现清除按钮（X 图标），点击可清除搜索条件并恢复完整列表。

搜索功能基于 SQLite FTS5 全文搜索引擎，使用 trigram 分词技术，支持中文子串匹配。搜索结果按相关性排序（BM25 算法），标题匹配权重高于内容匹配，确保最相关的笔记排在前面。

### 6.7 笔记设置

在编辑模式下，点击工具栏的"设置"按钮（齿轮图标）可打开笔记设置对话框，用于管理笔记的归属、标签和 AI 访问权限：

- **所属笔记本：** 通过可搜索的下拉选择器，将笔记移动到指定笔记本。
- **标签：** 以网格形式展示所有可用标签，以勾选框形式选中或取消关联的标签。一条笔记可关联多个标签。
- **AI 访问权限：** 设置 AI 工具通过 MCP 协议对该笔记的访问级别。可选值包括"继承"（默认，从标签和笔记本继承）、"读写"、"只读"、"禁止"。笔记级设置优先级最高，可以覆盖笔记本和标签的设置。详见 [20.4 权限控制](#204-权限控制)。

修改完成后点击"保存"按钮生效。

### 6.8 笔记置顶

用户可以将重要笔记置顶，使其始终显示在笔记列表的最顶部。

**操作方式：**

- 在笔记列表中，右键点击笔记卡片或点击卡片上的置顶图标按钮，切换笔记的置顶状态。
- 已置顶的笔记在卡片上显示置顶标识。
- 再次点击可取消置顶。

置顶笔记在列表中排列在所有非置顶笔记之前，多条置顶笔记之间按更新时间排序。

### 6.9 笔记加密

ENote 支持对笔记内容进行 AES-256-GCM 加密，保护敏感信息的安全。

**加密笔记：**

1. 选中要加密的笔记，确保处于编辑模式。
2. 在工具栏中点击"加密"按钮。
3. 在弹出的加密对话框中输入密码，并再次确认密码。
4. 点击"加密"按钮，笔记内容将被加密保存。

**解密笔记：**

1. 打开一条已加密的笔记（编辑器会显示"此笔记内容已加密"的提示）。
2. 点击"解密"按钮。
3. 在弹出的对话框中输入加密时设置的密码。
4. 密码正确后，笔记内容恢复为明文显示。

> **注意：**
> - 加密使用 AES-256-GCM 算法，密钥通过 SHA-256 从用户密码派生，安全性高。
> - 请务必牢记加密密码，丢失密码将无法恢复笔记内容。
> - 加密后的笔记内容以密文形式存储在数据库中，无法通过搜索找到其原始内容。

### 6.10 关联笔记（双向链接）

ENote 支持在笔记之间建立双向链接关系，方便构建知识网络。

**查看关联笔记：**

在编辑器底部的"关联笔记"面板中，点击面板标题栏可展开/折叠面板。面板显示与当前笔记关联的所有笔记列表，包括笔记标题。点击任一关联笔记的标题可快速跳转查看该笔记。

**添加关联：**

1. 在关联笔记面板中，点击"添加链接"按钮。
2. 在弹出的搜索框中输入要关联的笔记标题关键词。
3. 从搜索结果列表中点击选择目标笔记，链接即刻建立。

**移除关联：**

将鼠标悬停在某条关联笔记上，点击右侧出现的"×"按钮即可移除该链接。

> **注意：** 双向链接是对称的——如果笔记 A 链接到笔记 B，那么在笔记 B 的关联面板中也会显示笔记 A。删除任一端的链接会同时移除双方的关联。

### 6.11 在新窗口中打开笔记

ENote 支持在独立窗口中打开笔记，方便同时查看或对比多条笔记。

**操作方式：** 通过命令面板或工具栏中的"在新窗口中打开"选项，将当前笔记在一个独立的应用窗口中打开。新窗口可独立调整大小和位置，具备完整的编辑功能。

## 7. 富文本编辑器

ENote 集成了功能丰富的所见即所得（WYSIWYG）富文本编辑器，工具栏位于编辑器顶部，当工具栏按钮过多时支持水平滚动，左右两侧有滚动指示箭头。

### 7.1 文本样式

工具栏提供以下文本格式化按钮：

| 按钮 | 功能 | 说明 |
|------|------|------|
| **B** | 粗体 | 将选中文本设为粗体 |
| *I* | 斜体 | 将选中文本设为斜体 |
| U | 下划线 | 为选中文本添加下划线 |
| ~~S~~ | 删除线 | 为选中文本添加删除线 |
| X² | 上标 | 将选中文本设为上标 |
| X₂ | 下标 | 将选中文本设为下标 |

点击按钮可切换对应格式的开启/关闭状态，选中文本后点击按钮应用格式，已应用的格式按钮会高亮显示。

### 7.2 标题与字体

**标题层级：** 通过下拉选择器设置段落的标题级别，支持：
- 正文（默认段落）
- 标题 1（H1）至 标题 6（H6），字号从大到小递减。

**字体族：** 下拉选择器提供多种字体选择，按类别分组：
- **无衬线体：** Arial、Helvetica、Verdana、Tahoma、Trebuchet MS、微软雅黑、苹方
- **衬线体：** Times New Roman、Georgia、Palatino、宋体、楷体、仿宋
- **等宽字体：** Courier New、Consolas、Monaco、Source Code Pro
- **艺术字体：** Comic Sans MS、Impact、Brush Script

**字体大小：** 下拉选择器支持 12px、14px、16px、18px、20px、24px、28px、32px、36px、48px、72px 等多种预设字号。

### 7.3 文本对齐

提供四种段落对齐方式按钮：

- **左对齐：** 文本靠左排列（默认）。
- **居中对齐：** 文本居中排列。
- **右对齐：** 文本靠右排列。
- **两端对齐：** 文本两端对齐排列。

### 7.4 列表功能

支持三种列表类型：

- **无序列表：** 以项目符号标记的列表。
- **有序列表：** 以数字编号标记的列表。
- **任务列表：** 带有复选框的任务清单列表，点击复选框可切换完成状态。已完成的任务项显示删除线和灰色文字效果。支持任务列表嵌套。

### 7.5 缩进控制

- **增加缩进：** 点击增加缩进按钮或按 Tab 键，段落向右缩进一级（每级 2em）。
- **减少缩进：** 点击减少缩进按钮或按 Shift+Tab 键，段落向左减少一级缩进。
- 最大缩进层级为 10 级，最小为 0 级。

### 7.6 引用与代码

- **引用块：** 点击引用按钮，将段落转为引用块样式，左侧显示主题色竖线，文本以灰色斜体显示。
- **代码块：** 点击代码块按钮，插入带语法高亮的代码块。支持多种编程语言的语法着色显示（基于 highlight.js 引擎）。
- **行内代码：** 点击行内代码按钮，将选中文本设为等宽字体的行内代码样式。

### 7.7 链接与图片

**插入链接：**
1. 选中需要添加链接的文本。
2. 点击工具栏的链接按钮。
3. 在弹出的对话框中输入链接地址（URL）。
4. 点击"确定"插入链接。链接文本以蓝色下划线样式显示。

**取消链接：** 选中已有链接的文本，点击"取消链接"按钮移除超链接。

**插入图片：**
1. 点击工具栏的图片按钮。
2. 在弹出的对话框中选择插入方式：
   - **输入 URL：** 在文本框中输入图片的网络地址。
   - **上传文件：** 点击上传按钮选择本地图片文件，支持常见图片格式。
3. 点击"确定"插入图片。

图片在编辑器中以圆角样式显示，支持懒加载技术，加载过程中显示动画占位符。

### 7.8 表格功能

- **插入表格：** 点击插入表格按钮，自动插入一个 3 行 3 列的表格。
- **添加列：** 在当前列后方插入新列。
- **添加行：** 在当前行下方插入新行。
- **删除表格：** 删除整个表格。
- 表格支持通过拖拽列边框调整列宽。选中的单元格会高亮显示。

### 7.9 颜色设置

**文字颜色：** 点击文字颜色按钮打开颜色选择器，提供 10 种预设颜色的宫格选择，同时支持通过颜色选择器或输入十六进制色值自定义颜色。

**背景颜色：** 点击背景颜色按钮，为选中文本设置背景高亮色，颜色选择方式同文字颜色。

**荧光笔高亮：** 点击高亮按钮，快速为选中文本添加黄色高亮背景。

### 7.10 查找与替换

点击工具栏的"查找替换"按钮，在编辑器上方打开查找替换面板：

- **查找输入框：** 输入要查找的文本，按回车键定位到下一个匹配项。
- **替换输入框：** 输入替换后的文本。
- **区分大小写：** 复选框控制是否区分大小写匹配。
- **结果计数：** 显示当前匹配位置和总匹配数（如"3 / 12"）。
- **上一个/下一个：** 箭头按钮在匹配结果间导航。
- **替换：** 替换当前匹配项。
- **全部替换：** 一次性替换所有匹配项。

所有匹配项在编辑器中以黄色高亮显示，当前聚焦的匹配项以橙色高亮区分。

### 7.11 目录导航

点击工具栏的"目录"按钮，在编辑器右侧打开浮动目录导航面板（宽度 240 像素）：

- 自动提取文档中的所有标题（H1-H6），按层级缩进展示。
- 点击目录中的任一条目，编辑器自动滚动定位到对应的标题位置。
- 当前可视区域内的标题在目录中高亮显示（左侧显示主题色竖线）。
- 点击面板右上角的关闭按钮（X）关闭目录面板。

### 7.12 撤销与重做

- **撤销：** 点击撤销按钮，回退到上一步编辑操作。
- **重做：** 点击重做按钮，恢复被撤销的操作。

### 7.13 清除格式

点击工具栏的"清除格式"按钮，移除选中文本的所有格式化样式，恢复为默认文本格式。

### 7.14 内容块拖拽排序

富文本编辑器支持通过拖拽方式调整内容块的顺序。将鼠标悬停在段落、标题、引用块、代码块等块级元素上时，元素左侧会出现拖拽手柄。按住拖拽手柄并拖动，可将该内容块移动到文档中的其他位置。拖动过程中，编辑器会显示当前拖拽状态的视觉指示，释放鼠标后内容块将插入到目标位置。

此功能适用于快速调整文档结构，无需通过剪切粘贴操作即可重新排列内容。

### 7.15 数学公式

富文本编辑器支持数学公式的渲染显示，基于 KaTeX 数学排版引擎。用户可以在笔记中插入数学公式，系统自动将公式语法渲染为标准的数学符号格式，支持行内公式和独立公式块，满足学术笔记和技术文档的数学表达需求。

### 7.16 图片懒加载

编辑器中的图片采用智能懒加载技术。当笔记包含较多图片时，仅加载当前可视区域内的图片，未进入可视区域的图片显示为带有动画效果的骨架占位符。当用户滚动页面使图片进入可视范围时，图片自动开始加载并以渐入动画显示。此机制有效减少初始加载时间，提升长文档的浏览流畅度。

### 7.17 智能粘贴

编辑器具备智能粘贴功能，自动清理从外部来源（如 Microsoft Office、Google Docs、网页等）复制粘贴的 HTML 内容：

- **清理 Office 标记：** 自动移除 MS Office 特有的 XML 命名空间、`mso-` 样式前缀等冗余标记。
- **清理网页格式：** 移除多余的 CSS 类名、`data-*` 属性，仅保留基本的文本格式（粗体、斜体、颜色等）。
- **标签转换：** 将 `<div>` 等块级元素自动转换为标准的 `<p>` 段落。
- **去除空元素：** 自动移除无内容的空段落和空 `<span>` 标签。
- **截图粘贴：** 支持直接粘贴剪贴板中的截图图片，图片自动保存为本地文件。

### 7.18 图片本地存储

编辑器中通过粘贴或拖拽插入的图片会自动保存为本地文件，而非以 Base64 编码内联存储在笔记内容中。图片文件保存在应用数据目录下的 `images/` 子目录中，每张图片使用 UUID 作为文件名，支持 PNG、JPG、GIF、WebP、SVG、BMP、ICO 等常见格式。

本地文件存储的优势：
- **减小数据库体积：** 笔记内容不再包含大量 Base64 编码的图片数据。
- **提升加载性能：** 图片通过 Tauri asset protocol 直接从本地文件系统加载，速度更快。
- **便于管理：** 图片文件独立存储，可直接在文件系统中查看和管理。

## 8. Markdown 编辑器

当笔记内容类型选择为 Markdown 时，编辑器提供专业的 Markdown 编写体验。

### 8.1 源码编辑模式

在源码模式下，编辑区域显示为深色主题的等宽字体文本编辑区，用户直接编写 Markdown 语法文本。支持标准 Markdown 语法，包括标题、列表、链接、图片、代码块、表格、引用等。

### 8.2 预览模式

点击工具栏的"源码/预览切换"按钮，可在 Markdown 源码编辑和渲染预览之间切换。预览模式下显示 Markdown 语法渲染后的格式化内容。

Markdown 预览支持以下增强渲染：
- **数学公式：** 基于 KaTeX 引擎渲染数学公式。使用 `$...$` 编写行内公式，使用 `$$...$$` 编写独立公式块。
- **语法高亮：** 代码块自动识别编程语言并着色显示。
- **表格、任务列表：** 标准 Markdown 扩展语法均可正常渲染。

### 8.3 分屏模式

Markdown 编辑器支持两种分屏布局，方便同时查看源码和预览效果：

**垂直分屏：** 点击工具栏的"垂直布局"按钮，编辑区上方显示 Markdown 源码，下方显示渲染预览。

**水平分屏：** 点击工具栏的"水平布局"按钮，编辑区左侧显示 Markdown 源码，右侧显示渲染预览。

分屏模式特性：
- 中间分隔线支持拖拽调整两个面板的比例（范围 20%-80%）。
- 源码区和预览区支持同步滚动，编辑时预览自动跟随定位。
- 大文档使用 Web Worker 技术进行异步预处理，确保编辑流畅。

ENote 自动记录笔记的每次修改和删除操作历史，支持版本追溯和内容对比。

### 9.1 查看历史列表

**操作步骤：**

1. 选中一条笔记。
2. 点击工具栏右侧的"历史记录"按钮（时钟图标）。
3. 系统弹出全屏历史记录对话框，以表格形式展示历史记录列表。

历史记录表格包含以下列信息：

## 10. 笔记模板

ENote 提供笔记模板功能，允许用户创建和管理常用的笔记模板，快速生成具有预设内容结构的新笔记。

### 10.1 管理模板

打开"模板管理"对话框有以下方式：

- **侧边栏：** 点击侧边栏底部工具栏中的"模板管理"图标按钮（模板图标）。
- **命令面板：** 按 Ctrl+P 打开命令面板，搜索"模板管理"并执行。

模板管理对话框采用两级界面设计：

**第一级 — 模板列表：**

- **查看模板列表：** 对话框中列出所有已创建的模板，显示模板名称和更新时间。
- **新建模板：** 点击顶部的"+ 新建模板"按钮，进入模板编辑界面。
- **编辑模板：** 点击模板右侧的"编辑"按钮，进入模板编辑界面修改名称和内容。
- **删除模板：** 点击模板右侧的"删除"按钮，删除不再需要的模板。

**第二级 — 模板编辑：**

- **返回按钮：** 点击左上角的返回箭头，返回模板列表。
- **模板名称与内容类型：** 上方一行包含名称输入框和内容类型切换按钮（富文本 / Markdown），与笔记编辑器的双模式一致。新建模板时默认为富文本模式，编辑已有模板时自动恢复其原有内容类型。切换内容类型时当前编辑内容自动保留。
- **编辑工具栏（富文本模式）：** 富文本模式下显示精简版格式化工具栏，包含标题层级（正文/H1/H2/H3）、文本样式（粗体、斜体、下划线、删除线）、列表（无序、有序、任务列表）、块级元素（引用块、代码块、水平线）、撤销/重做等常用按钮。
- **Markdown 模式：** 选择 Markdown 后，编辑区域切换为 Markdown 源码编辑模式，不显示格式化工具栏，用户直接编写 Markdown 语法。
- **保存：** 点击底部的"保存"按钮保存模板，模板的内容类型同时保存。

> **注意：** 模板列表中每个模板名称旁会显示内容类型徽章：**RT**（富文本）或 **MD**（Markdown），便于快速识别。

### 10.2 使用模板创建笔记

在模板管理对话框中，点击某个模板右侧的"使用模板"按钮，系统将以该模板的内容为基础创建一条新笔记。新笔记自动继承模板的内容类型（富文本或 Markdown），并进入编辑模式，用户可在模板内容基础上继续编辑。

也可通过命令面板搜索"从模板新建笔记"快速使用模板创建笔记。

### 10.3 将笔记保存为模板

在编辑笔记时，点击编辑器工具栏右侧操作按钮区的"保存为模板"按钮（模板图标），即可将当前笔记的内容和内容类型一并保存为模板。也可通过命令面板搜索"保存为模板"执行此操作。保存为模板后，原笔记不受影响。

## 11. 导出功能

ENote 支持将笔记导出为多种格式，满足不同的使用场景和数据迁移需求。

**操作步骤：**

1. 选中要导出的笔记。
2. 点击工具栏右侧的"导出"按钮（下载图标）。
3. 在弹出的导出对话框中选择目标格式。
4. 点击"导出"按钮。
5. 系统弹出文件保存对话框，选择保存位置和文件名。
6. 导出完成后显示成功通知。

### 11.1 导出为 Word 文档

将笔记导出为 .doc 格式的 Word 文档，保留文本格式和样式。文档顶部包含笔记标题作为主标题，支持在 Microsoft Word、WPS 等办公软件中打开编辑。

### 11.2 导出为 Markdown

将笔记导出为 .md 格式的 Markdown 文件。富文本类型的笔记会自动转换为 Markdown 语法；Markdown 类型的笔记直接导出原始内容。

### 11.3 导出为印象笔记格式

将笔记导出为 .enex 格式，兼容印象笔记（Evernote）的导入功能，便于将笔记迁移至印象笔记平台。

### 11.4 导出为 JSON

将笔记导出为 .json 格式的结构化数据文件，包含以下字段：
- 标题（title）
- 内容（content）
- 内容类型（contentType）
- 标签（tags）
- 创建时间（createTime）
- 更新时间（updateTime）

### 11.5 导出为 XML

将笔记导出为 .xml 格式，数据结构与 JSON 导出一致，适用于需要 XML 格式数据交换的场景。

## 13. 数据备份与恢复

ENote 提供完整的数据备份与恢复功能，支持手动导出和自动定时备份两种方式，确保数据安全。

点击侧边栏底部的"数据备份"图标按钮（数据库图标）打开数据备份对话框。对话框包含"导出备份"和"导入恢复"两个选项卡。

### 13.1 导出备份

**操作步骤：**

1. 打开数据备份对话框，默认处于"导出备份"选项卡。
2. 选择备份格式：
   - **SQL：** 导出为标准 SQL 语句文件（.sql），适合跨数据库迁移。
   - **Excel：** 导出为 Excel 表格（.xlsx），每张数据表对应一个 Sheet。
   - **CSV：** 导出为 CSV 文件，打包为 ZIP 压缩包（.zip）。
3. 点击"导出"按钮。
4. 在系统文件保存对话框中选择保存位置和文件名。
5. 导出完成后显示成功通知。

### 13.2 导入恢复

**操作步骤：**

1. 打开数据备份对话框，切换到"导入恢复"选项卡。
2. 选择与备份文件对应的格式（SQL / Excel / CSV）。
3. 点击"导入"按钮。
4. 在系统文件选择对话框中选择备份文件。
5. 系统弹出确认对话框，提示导入操作将清除当前所有数据并替换为备份数据。
6. 点击"确认导入"执行恢复，或点击"取消"放弃操作。
7. 恢复完成后显示成功通知。

> **注意：** 导入操作会覆盖当前所有数据，此操作不可撤销。执行导入前请确保已备份当前数据。导入完成后建议刷新页面以加载最新数据。

### 13.3 自动备份

ENote 支持自动定时备份功能，在后台按设定的时间间隔自动创建数据库备份，无需手动操作。

**工作原理：**

- 自动备份以 SQL 格式保存到应用数据目录下的 `backups/` 子目录中。
- 备份文件命名格式为 `enote_backup_YYYYMMDD_HHMMSS.sql`，包含完整的时间戳。
- 系统会根据设定的保留份数自动清理旧备份，避免占用过多磁盘空间。
- 应用启动时会自动检查是否需要执行备份（距上次备份是否超过设定间隔）。

**配置方式：** 在"设置"对话框的"自动备份"区域进行配置，详见 [14.5 自动备份设置](#145-自动备份设置)。

## 14. 应用设置

点击侧边栏底部的"设置"图标按钮打开设置对话框，可配置以下选项。

### 14.1 主题设置

ENote 支持三种主题模式：

- **浅色模式：** 经典的白色背景界面，适合日间使用。
- **深色模式：** 深色背景界面，减少眼部疲劳，适合夜间或低光环境使用。
- **跟随系统：** 自动跟随操作系统的深浅色设置，系统切换主题时应用界面实时跟随变化。

在设置对话框的"外观"区域，点击对应的主题按钮即可切换。主题切换即时生效，设置自动保存。

### 14.2 语言设置

ENote 支持中文简体（zh-CN）和英文（en-US）两种界面语言。

在设置对话框的"外观"区域，点击对应的语言按钮即可切换。切换后界面所有文本立即更新，无需重启应用。语言偏好设置会自动保存，下次启动时自动应用。

### 14.3 布局模式设置

ENote 支持根据窗口宽度自动适配布局，也可手动指定布局模式：

| 模式 | 说明 |
|------|------|
| **自动（默认）** | 根据窗口宽度自动选择：> 1024px 桌面模式，640-1024px 平板模式，< 640px 手机模式 |
| **桌面** | 强制使用三栏并排布局 |
| **平板** | 强制使用双栏布局（侧边栏覆盖层） |
| **手机** | 强制使用单视图全屏切换布局 |

在设置对话框的"外观"区域，点击对应的布局按钮即可切换。也可使用快捷键 Ctrl+Shift+M（macOS 为 Cmd+Shift+M）循环切换，或通过命令面板搜索"切换布局"执行。

### 14.4 快捷键设置

在设置对话框的"快捷键"区域，可自定义应用级键盘快捷键。

**系统级快捷键**（如 Ctrl+S 保存、Esc 取消编辑）为固定绑定，无法修改，以"系统"标签标识。

**可自定义快捷键**的操作方式：

1. 点击快捷键显示区域，进入录制状态，界面提示"请按下快捷键..."。
2. 按下新的快捷键组合（如 Ctrl+K），录制完成后自动保存。
3. 如果新快捷键与已有快捷键冲突，显示红色冲突提示，不会保存。
4. 按 Esc 取消录制。

每个自定义快捷键右侧有「重置」按钮，可恢复为默认值。顶部有「全部重置为默认」按钮，一键恢复所有快捷键。

快捷键设置自动持久化，重启应用后仍然有效。命令面板中显示的快捷键文本会同步更新。

### 14.5 自动备份设置

在设置对话框的"自动备份"区域，可配置以下选项：

| 选项 | 说明 |
|------|------|
| **启用自动备份** | 开关控件，开启或关闭自动备份功能 |
| **备份间隔** | 下拉选择备份频率：1小时、4小时、8小时、24小时 |
| **保留份数** | 下拉选择保留的最大备份数量：最近5份、10份、20份、50份 |
| **立即备份** | 手动触发一次备份操作 |
| **上次备份** | 显示最近一次自动备份的文件名和时间 |

启用自动备份后，系统在后台按设定间隔自动执行备份，并根据保留份数自动清理旧备份文件。

### 14.6 安全设置（锁屏）

ENote 提供锁屏安全功能，可在应用启动时或一段时间无操作后自动锁定界面，需要输入密码方可继续使用，防止未经授权的访问。

**默认状态：** 锁屏功能默认关闭，用户可根据需要手动开启。

在设置对话框的"安全"区域，可配置以下选项：

| 选项 | 说明 |
|------|------|
| **锁屏模式** | 切换锁屏保护的开启/关闭状态。可选"关闭"或"密码" |
| **设置密码** | 首次开启时设置锁屏密码（至少 4 位） |
| **修改密码** | 已设置密码时，可输入新密码替换当前密码 |
| **移除密码** | 清除已设置的锁屏密码并关闭锁屏功能 |
| **超时锁定** | 设置无操作自动锁定的等待时间：仅启动时、1分钟、5分钟、15分钟、30分钟 |
| **最小化时锁定** | 开关控件，开启后应用窗口最小化或切换到后台时自动锁定 |

**锁屏界面：**

当应用被锁定时，显示全屏半透明遮罩层和居中的锁屏卡片。用户需输入正确的锁屏密码并点击"解锁"按钮方可进入应用。密码错误时显示错误提示并播放抖动动画。

**手动锁定：** 按 Ctrl+L（macOS 为 Cmd+L）快捷键可随时手动锁定应用，也可通过命令面板搜索"锁定应用"执行。

**密码安全：** 锁屏密码使用 Argon2id 算法进行哈希处理后存储，不以明文形式保存，确保密码安全。

### 14.7 MCP 设置

MCP（Model Context Protocol）是一项 AI 工具集成功能，允许 Claude Desktop、Claude Code 等 AI 客户端通过标准协议操作笔记数据。MCP 功能默认关闭，需在设置中手动启用。

在设置对话框的"MCP (AI 工具集成)"区域，可配置以下选项：

| 选项 | 说明 |
|------|------|
| **启用 MCP** | 总开关，控制是否允许 AI 工具通过 MCP 协议操作笔记 |
| **搜索笔记** | 允许 AI 工具搜索笔记内容 |
| **获取笔记详情** | 允许 AI 工具读取笔记完整内容 |
| **创建笔记** | 允许 AI 工具创建新笔记 |
| **更新笔记** | 允许 AI 工具修改已有笔记 |
| **删除笔记** | 允许 AI 工具删除笔记（移入回收站） |
| **列出笔记本** | 允许 AI 工具查看笔记本列表 |
| **创建笔记本** | 允许 AI 工具创建新笔记本 |
| **列出标签** | 允许 AI 工具查看标签列表 |
| **创建标签** | 允许 AI 工具创建新标签 |
| **笔记统计** | 允许 AI 工具获取笔记数量统计 |

启用总开关后，展开各工具的独立开关列表，可精细控制每个 MCP 工具的可用性。设置实时生效，无需重启 MCP Server。

> **注意：** MCP 设置控制 AI 工具的全局和工具级访问权限。禁用某个工具后，AI 客户端将无法看到和调用该工具。通过 AI 工具创建或修改的笔记，在历史记录中会标记为"AI 工具"来源，便于区分人工操作和 AI 操作。
>
> 除了全局和工具级控制外，ENote 还支持更细粒度的**数据级访问控制**——可以在笔记本、标签和单条笔记上分别设置 AI 的访问权限（继承/读写/只读/禁止），详见 [20.4 权限控制](#204-权限控制)。

### 14.8 配置管理

在设置对话框中，**配置管理** 区块提供以下功能：

- **当前配置信息：** 显示当前使用的 Profile 名称和数据库类型。
- **切换配置：** 点击「切换配置」按钮，返回配置选择页面。在配置选择页面中可以：
  - 选择已有的数据库配置并连接
  - 点击「新建配置」进入设置向导，创建新的数据库配置

### 14.9 系统维护（跨 Profile 同步）

在设置对话框底部的"系统维护"区域，提供跨 Profile 数据同步和同步记录管理功能。

#### 14.9.1 跨 Profile 同步

将当前 Profile 的数据同步到其他 Profile，支持跨数据库类型同步（SQLite ↔ MySQL ↔ PostgreSQL）。

**操作步骤：**

1. 在"系统维护"区域，点击「开始同步」按钮打开同步对话框。
2. 选择目标 Profile（下拉列表显示除当前 Profile 外的所有可用配置）。
3. 如果目标 Profile 使用密码认证（MySQL/PostgreSQL），需输入目标数据库密码。
4. 选择同步模式：

| 模式 | 说明 |
|------|------|
| **追加（默认）** | 将数据追加到目标数据库，不影响目标已有数据。ID 由目标库自增分配 |
| **覆盖** | 清空目标数据库后全量写入。选择此模式需二次确认 |

5. 勾选同步范围（可多选）：

| 数据类型 | 说明 |
|---------|------|
| **笔记本** | 笔记本及层级结构 |
| **标签** | 所有标签 |
| **笔记** | 笔记内容、标签关联、置顶状态、回收站状态 |
| **笔记历史** | 源端的原始历史记录 |
| **模板** | 笔记模板 |
| **设置** | 应用设置（键值对） |

6. 备份设置：建议开启「同步前自动备份」，可选择备份格式（SQL / Excel / CSV）。
7. 选择目标 Profile 后，下方显示源端和目标端的数据量预览。
8. 点击「开始同步」执行同步，界面显示实时进度（各表同步进度 + 总进度条）。

**关键行为：**

- **加密处理：** 如果源端和目标端的加密配置不同，系统自动处理——从源端解密读取，向目标端重新加密写入，确保数据一致性。
- **历史记录：** 笔记同步通过 service 层执行，目标端自动生成同步来源的历史记录（操作来源标记为"同步"）。
- **外键映射：** 笔记本和标签的 ID 在目标库中重新分配，笔记的笔记本归属和标签关联自动修正。

#### 14.9.2 同步记录管理

每次同步操作自动生成详细记录，包含每条数据的同步状态。

点击"系统维护"区域的「查看详情」按钮打开同步记录管理对话框。

**同步记录列表：** 显示所有历史同步操作，包含以下信息：

| 字段 | 说明 |
|------|------|
| 源 → 目标 | 源 Profile 和目标 Profile 名称 |
| 状态 | 完成 / 失败 |
| 成功/失败数 | 成功和失败的记录条数 |
| 同步模式 | 追加 / 覆盖 |
| 数据库类型 | 例如 sqlite → mysql |
| 时间 | 同步执行的时间 |

**操作：**

- **查看详情：** 展开单次同步的完整结果，包括汇总表格和逐条明细。明细支持按表名和状态筛选。
- **导出日志：** 将同步记录导出为 JSON 文件，便于排查问题。
- **删除记录：** 删除单条同步记录及其明细。
- **清空所有记录：** 清除全部历史同步记录。

## 19. 应用启动与初始化

### 19.1 命令行参数

ENote 支持通过命令行参数指定配置文件路径，适用于开发调试或多环境部署场景：

```bash
# 使用自定义配置文件启动（兼容旧模式）
enote --config /path/to/application.yml
enote -c /path/to/application.yml
```

- 支持绝对路径和相对路径（相对路径基于当前工作目录解析）。
- 使用 `--config` 参数时将跳过 Profile 模式，直接使用指定的配置文件。
- 未指定 `--config` 参数时，进入 Profile 模式（详见下文）。

### 19.2 首次启动（设置向导）

首次启动 ENote 时，系统检测到尚无任何 Profile 配置，将自动显示 **设置向导** 界面，引导用户完成数据库连接配置。

设置向导包含以下步骤：

**步骤一：选择数据库类型**

- **SQLite（推荐）：** 本地文件数据库，无需安装额外服务，开箱即用。适合个人使用。
- **MySQL：** 网络数据库，适合团队协作或远程访问场景。需预先安装 MySQL 5.7+。
- **PostgreSQL：** 功能丰富的企业级数据库。需预先安装 PostgreSQL 12+。

**步骤二：配置连接信息**

根据所选的数据库类型，填写相应的连接参数：

- **配置名称：** 为此配置取一个便于识别的名称（如「本地笔记」、「工作数据库」）。
- **SQLite：** 选择数据库文件存储路径。支持「新建」（选择保存位置创建新数据库）和「打开已有」（选择已存在的 .db 文件）两种方式。留空则使用默认位置（应用数据目录）。
- **MySQL / PostgreSQL：** 填写主机地址、端口、数据库名，选择认证方式：
  - **密码登录：** 输入用户名和密码。密码将安全存储在操作系统钥匙串中，不保存在配置文件里。
  - **证书登录：** 输入用户名，选择 SSL 模式（优先/必须/验证 CA/验证身份），并指定 CA 证书、客户端证书、客户端私钥文件路径。

可点击「测试连接」按钮验证配置是否正确。

**步骤三：安全设置**

- **内容加密：** 开关控制是否启用笔记内容加密。启用后：
  - 系统自动生成 AES-256 加密密钥，安全存储在操作系统钥匙串（macOS Keychain / Windows Credential Store / Linux Secret Service）中。
  - 所有笔记内容在写入数据库前自动加密，读取时自动解密，对用户完全透明。
  - 即使数据库文件被直接访问，笔记内容也无法被读取。
  - **注意：** 启用加密后，笔记标题仍可搜索，但内容将无法通过全文搜索检索。

**步骤四：保存并连接**

确认配置后，系统自动保存配置文件和密钥，连接数据库并执行初始化，然后重启应用进入主界面。

> **语言切换：** 设置向导右上角提供语言切换按钮（🌐 图标），点击可在中文和英文之间切换，方便非中文用户首次使用。

> **窗口关闭：** 首次启动时，设置向导为必须完成的步骤，不提供跳过或取消按钮。如需退出，可直接关闭窗口（点击标题栏 ✕ 按钮），应用将直接退出。

### 19.3 多配置选择

当系统中存在多个 Profile 配置时，启动后将显示 **配置选择页面**：

- 以卡片形式展示所有配置，显示名称、数据库类型、连接信息、加密状态等。
- 上次使用的配置会标记「上次使用」标签。
- 双击或选中后点击「连接」按钮即可使用该配置启动。
- 点击「新建配置」进入设置向导，创建新的数据库配置。
- 支持编辑、删除已有配置。
- 勾选「下次自动连接上次使用的配置」可跳过选择页面，直接使用上次的配置启动。

仅有一个配置时，或开启了自动连接时，将跳过选择页面直接启动。

> **关闭行为：** 首次启动时的配置选择页面无关闭按钮（必须选择一个配置）。从主界面通过设置进入的配置选择页面提供 ✕ 关闭按钮，可返回主界面继续使用当前配置。

### 19.4 启动异常处理

若启动过程中遇到异常，系统会弹出操作系统原生错误对话框，提示具体的错误信息：

- **配置文件加载失败：** 提示配置文件读取错误，请检查配置文件格式是否正确。
- **数据库连接失败：** 提示数据库连接异常，请检查数据库配置信息或数据库服务是否可用。连接失败时系统将自动回退到设置向导，允许用户重新配置。
- **数据库迁移失败：** 提示数据表结构更新异常。

错误提示信息支持中英文国际化，根据系统语言环境自动显示对应语言。

### 19.5 后续启动

非首次启动时，系统自动检测并执行数据库结构升级（如有新版本的表结构变更），确保数据兼容性。整个过程对用户透明，无需手动干预。启动时还会自动检查自动备份设置，如已启用且距上次备份超过设定间隔，则自动执行一次备份。

## 20. MCP（AI 工具集成）

### 20.1 功能概述

MCP（Model Context Protocol，模型上下文协议）是 Anthropic 提出的开放协议，允许 AI 助手与外部工具和数据源交互。ENote 内置了 MCP Server，使 Claude Desktop、Claude Code、Cursor 等 AI 客户端能够直接读写笔记数据。

通过 MCP 集成，AI 助手可以：

- 搜索和阅读笔记内容，获取知识上下文
- 根据对话创建新笔记，自动整理信息
- 更新已有笔记，补充或修改内容
- 管理笔记本和标签，组织知识结构

所有通过 MCP 进行的写操作都会记录在笔记历史中，并标注为"AI 工具"来源，确保可审计和可回滚。

### 20.2 配置与启用

MCP 功能的启用需要两个步骤：

**步骤一：配置文件启用**

在 `application.yml` 配置文件中启用 MCP：

```yaml
# MCP Server 配置
mcp:
  enabled: true
```

**步骤二：构建 MCP Server**

```bash
cd src-tauri
cargo build --release --bin enote-mcp
```

构建完成后，在 `target/release/` 目录下生成 `enote-mcp` 可执行文件。

**步骤三：配置 AI 客户端**

以 Claude Desktop 为例，在 `claude_desktop_config.json` 中添加：

```json
{
  "mcpServers": {
    "enote": {
      "command": "/path/to/enote-mcp",
      "args": ["--config", "/path/to/application.yml"]
    }
  }
}
```

Claude Code 的配置方式类似，在 `.claude/settings.json` 中配置 MCP Server。

**步骤四：设置面板精细控制**

在 ENote 的设置面板（[14.7 MCP 设置](#147-mcp-设置)）中，启用 MCP 总开关并根据需要开启/关闭各个工具。

### 20.3 可用工具列表

MCP Server 提供以下 10 个工具：

| 工具名称 | 说明 | 类型 | 访问控制 |
|---------|------|------|---------|
| `search_notes` | 搜索笔记，支持关键词、笔记本和标签过滤，返回分页结果 | 读取 | 自动过滤"禁止"的笔记 |
| `get_note` | 获取指定笔记的完整内容（含 HTML、标签、笔记本信息） | 读取 | 检查读取权限 |
| `create_note` | 创建新笔记，可指定标题、内容、笔记本和标签 | 写入 | 检查目标笔记本写入权限 |
| `update_note` | 更新已有笔记的标题、内容、笔记本或标签 | 写入 | 检查笔记写入权限 |
| `delete_note` | 删除笔记（移入回收站，可在应用中恢复） | 写入 | 检查笔记写入权限 |
| `list_notebooks` | 列出所有笔记本（含 AI 访问权限字段） | 读取 | 元数据始终可见 |
| `create_notebook` | 创建新笔记本 | 写入 | — |
| `list_tags` | 列出所有标签（含 AI 访问权限字段） | 读取 | 元数据始终可见 |
| `create_tag` | 创建新标签 | 写入 | — |
| `note_stats` | 获取笔记统计信息（总数及各笔记本笔记数量） | 读取 | — |

> **安全说明：** 永久删除、清空回收站、加密/解密等危险操作不通过 MCP 暴露，防止 AI 工具误操作导致数据丢失。

### 20.4 权限控制

ENote 的 MCP 权限控制采用三层架构，从全局到数据级提供由粗到细的访问控制。

#### 20.4.1 三层访问控制架构

```
AI 请求 → ① 全局 MCP 开关 → ② 工具级开关 → ③ 数据级访问控制
           (设置面板)        (设置面板)        (mcp_access 字段)
```

| 层级 | 控制范围 | 设置位置 |
|------|---------|---------|
| **第一层：全局开关** | 控制 MCP 功能是否启用，关闭后所有 AI 工具不可用 | 设置 → MCP → 启用 MCP |
| **第二层：工具级开关** | 控制每个 MCP 工具（如搜索、创建、更新等）是否可用 | 设置 → MCP → 各工具开关 |
| **第三层：数据级访问控制** | 控制 AI 对具体笔记本/标签/笔记的读写权限 | 笔记本/标签编辑弹窗、笔记设置对话框 |

三层互补，不冲突。全局和工具级控制是"能不能用这个工具"，数据级控制是"能不能访问这条数据"。

工具级控制方面：

1. **`list_tools` 动态过滤：** AI 客户端调用 `list_tools` 时，MCP Server 从设置数据库读取已启用的工具列表，仅返回启用的工具。禁用的工具对 AI 不可见。

2. **调用时二次校验：** 每次工具调用前，MCP Server 再次检查该工具是否已启用，防止绕过。

设置面板的 MCP 开关修改即时生效——用户关闭某个工具后，AI 客户端下次调用 `list_tools` 就不会看到该工具。

#### 20.4.2 数据级访问控制（mcp_access）

笔记本、标签和笔记各有一个"AI 访问权限"（`mcp_access`）字段，支持四个级别：

| 值 | 名称 | 含义 |
|----|------|------|
| 继承 | Inherit | 从上层继承权限（默认值，向后兼容） |
| 读写 | ReadWrite | AI 可以读取和修改 |
| 只读 | ReadOnly | AI 只能读取，不能修改或删除 |
| 禁止 | Deny | AI 完全不可访问，搜索结果中也不可见 |

#### 20.4.3 权限解析规则

对于一条笔记，有效权限按以下优先级从高到低解析：

1. **加密笔记 → 强制禁止：** 加密笔记始终不允许 AI 访问，不可覆盖。
2. **笔记自身设置：** 如果笔记的 AI 访问权限不是"继承"，使用笔记自身的设置。笔记级设置可以**放宽**或**收紧**上层限制。
3. **标签设置：** 如果笔记关联了多个标签，取所有非"继承"标签中**最严格**的权限。
4. **笔记本设置：** 如果以上都是"继承"，使用笔记本的设置。若标签和笔记本同时有非"继承"设置，取两者中更严格的。
5. **默认读写：** 如果所有层级都是"继承"，则默认允许读写（全局 MCP 开关已经控制了总入口）。

**示例：**

| 笔记本设置 | 标签设置 | 笔记设置 | 有效权限 | 说明 |
|-----------|---------|---------|---------|------|
| 读写 | 只读 | 继承 | **只读** | 继承标签限制 |
| 禁止 | 读写 | 继承 | **禁止** | 笔记本禁止，取更严格值 |
| 禁止 | — | 读写 | **读写** | 笔记显式覆盖，放宽上层限制 |
| 只读 | — | 禁止 | **禁止** | 笔记显式覆盖，收紧上层限制 |
| 读写 | — | 读写（加密笔记） | **禁止** | 加密笔记始终禁止，不可覆盖 |

**元数据可见性说明：** `list_notebooks` 和 `list_tags` 始终返回所有条目（包含 `mcpAccess` 字段），AI 能看到笔记本和标签的名称及权限标记，但无法访问被禁止的笔记内容。

**各 MCP 工具的访问控制行为：**

| 工具 | 访问控制行为 |
|------|------------|
| `search_notes` | 搜索结果中自动过滤掉"禁止"的笔记，"只读"笔记正常返回 |
| `get_note` | 检查读取权限，"禁止"的笔记返回拒绝信息 |
| `create_note` | 检查目标笔记本的写入权限，"只读"或"禁止"的笔记本不允许创建 |
| `update_note` | 检查写入权限，"只读"或"禁止"的笔记拒绝更新 |
| `delete_note` | 检查写入权限，"只读"或"禁止"的笔记拒绝删除 |
| `list_notebooks` | 返回所有笔记本（含 `mcpAccess` 字段），不过滤 |
| `list_tags` | 返回所有标签（含 `mcpAccess` 字段），不过滤 |
| `note_stats` | 正常统计（统计值可能包含被禁止的笔记数量） |

#### 20.4.4 设置 AI 访问权限

**笔记本级别：**

1. 在侧边栏选中笔记本，点击菜单 → 编辑。
2. 在编辑弹窗底部找到"AI 访问权限"下拉框。
3. 选择所需的权限级别，点击保存。
4. 该笔记本下所有未单独设置权限的笔记将继承此设置。

**标签级别：**

1. 在侧边栏选中标签，点击菜单 → 编辑。
2. 在编辑弹窗底部找到"AI 访问权限"下拉框。
3. 选择所需的权限级别，点击保存。
4. 所有带该标签且未单独设置权限的笔记将受此设置影响。

**笔记级别：**

1. 在编辑模式下，点击工具栏的"设置"按钮（齿轮图标）。
2. 在笔记设置对话框底部找到"AI 访问权限"下拉框。
3. 选择所需的权限级别，点击保存。
4. 笔记级别的显式设置会覆盖笔记本和标签的设置。

> **提示：** 在侧边栏中，设置了非"继承"AI 访问权限的笔记本和标签旁会显示一个盾牌图标（🛡），方便快速识别哪些项目启用了 AI 访问控制。

### 20.5 操作来源追溯

笔记历史记录中新增了"操作来源"字段，区分操作的发起方：

| 来源 | 说明 |
|------|------|
| **用户操作** | 通过 ENote 界面进行的操作 |
| **AI 工具** | 通过 MCP 协议由 AI 客户端发起的操作 |

在历史记录表格中，操作来源与操作类型（新增/修改/删除）一同显示，方便用户识别哪些修改是 AI 做的，便于审计和回滚。

## 21. 数据存储与配置

### 21.1 本地数据存储

所有笔记数据默认存储在本地 SQLite 数据库中，数据完全保存在用户本地设备上，不经过任何网络传输，保障数据隐私安全。

应用配置文件存储在操作系统的应用数据目录中。使用默认 SQLite 数据库时，数据库文件也位于该目录下；使用 MySQL 或 PostgreSQL 时，数据存储在对应的数据库服务器上，该目录下仅保存配置文件。不同操作系统的应用数据目录路径如下：

| 操作系统 | 应用数据目录 | 示例 |
|----------|-------------|------|
| **Windows** | `{FOLDERID_RoamingAppData}\net.easycloudcn.enote\` | `C:\Users\Alice\AppData\Roaming\net.easycloudcn.enote\` |
| **macOS** | `$HOME/Library/Application Support/net.easycloudcn.enote/` | `/Users/Alice/Library/Application Support/net.easycloudcn.enote/` |
| **Linux** | `$XDG_DATA_HOME/net.easycloudcn.enote/` 或 `$HOME/.local/share/net.easycloudcn.enote/` | `/home/alice/.local/share/net.easycloudcn.enote/` |

> **快速访问：**
> - **Windows：** 在文件资源管理器地址栏输入 `%APPDATA%\net.easycloudcn.enote` 即可快速打开。
> - **macOS：** 在 Finder 中按 `Cmd+Shift+G`，输入 `~/Library/Application Support/net.easycloudcn.enote` 即可打开。
> - **Linux：** 在终端中执行 `xdg-open "${XDG_DATA_HOME:-$HOME/.local/share}/net.easycloudcn.enote"` 即可打开。

该目录下包含以下文件和子目录：

| 文件/目录 | 说明 |
|-----------|------|
| `profiles.yml` | Profile 索引文件，记录当前活跃的配置和自动连接设置 |
| `profiles/` | Profile 配置目录，每个数据库配置对应一个 YAML 文件 |
| `application.yml` | 旧版配置文件（兼容模式，使用 `--config` 参数时使用） |
| `enote.db` | SQLite 数据库文件，仅使用 SQLite 配置时存在 |
| `enote.db-wal` | SQLite WAL 日志文件（自动生成），仅 SQLite 时存在 |
| `enote.db-shm` | SQLite 共享内存文件（自动生成），仅 SQLite 时存在 |
| `images/` | 图片本地存储目录，保存编辑器中插入的图片文件 |
| `backups/` | 自动备份目录，保存定时备份的 SQL 文件 |

### 21.2 配置文件

应用配置文件为 `application.yml`，采用 YAML 格式，默认位于上述应用数据目录中。首次启动时系统会自动生成包含 SQLite 默认配置的文件，用户无需手动创建。也可通过命令行参数 `--config` 指定自定义配置文件路径（详见 [19.1 命令行参数](#191-命令行参数)）。

**默认生成的配置文件内容如下：**

```yaml
# ENote 应用配置文件
# 此文件由应用自动生成

datasource:
  # SQLite 数据库连接 URL
  # mode=rwc 表示读写模式，如果文件不存在则创建
  url: "sqlite:/path/to/net.easycloudcn.enote/enote.db?mode=rwc"

  # 连接池配置
  # 最大连接数：SQLite 推荐 5 以内
  max-connections: 5
  # 最小连接数：保持常驻连接减少重连开销
  min-connections: 1
  # 连接超时（秒）：建立新连接的超时时间
  connect_timeout: 10
  # 获取连接超时（秒）：从池中获取连接的超时时间
  acquire-timeout: 5
  # 空闲超时（秒）：空闲连接保持时间
  idle-time: 300
  # 最大生命周期（秒）：连接最长存活时间
  max-lifetime: 1800

# MCP Server 配置
# 启用后可通过 AI 工具（如 Claude Desktop）操作笔记
mcp:
  enabled: false
```

> **注意：**
> - 配置文件中 `url` 字段的实际路径会根据操作系统自动填入完整的绝对路径。SQLite URL 中支持使用 `~` 表示用户主目录（如 `sqlite://~/data/enote.db?mode=rwc`），系统会自动展开为完整路径。
> - 修改配置文件后需**重启应用**使配置生效。

### 21.3 配置项说明

以下为 `datasource` 节点下所有可配置项的详细说明：

| 配置项 | 类型 | 默认值 | 有效范围 | 说明 |
|--------|------|--------|----------|------|
| `url` | 字符串 | （自动生成） | — | **必填**。数据库连接 URL，格式取决于数据库类型（见 21.4 节） |
| `max-connections` | 整数 | SQLite: 5，其他: 20 | 1 - 1000 | 连接池最大连接数。SQLite 建议不超过 5 |
| `min-connections` | 整数 | 1 | 0 - max-connections | 连接池最小保持连接数，设为 0 表示不保持常驻连接 |
| `connect_timeout` | 整数 | 10 | 1 - 300 | 建立新数据库连接的超时时间（秒） |
| `acquire-timeout` | 整数 | 5 | 1 - 300 | 从连接池获取可用连接的超时时间（秒） |
| `idle-time` | 整数 | 300 | 1 - 86400 | 空闲连接保持时间（秒），超过此时间未使用的连接将被回收 |
| `max-lifetime` | 整数 | 1800 | 1 - 86400 | 连接最大存活时间（秒），超过此时间的连接将被强制关闭并重建 |

以下为 `mcp` 节点下的配置项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enabled` | 布尔值 | false | MCP Server 总开关。设为 true 后 MCP Server 才会接受连接 |

### 21.4 数据库配置示例

ENote 支持三种数据库，通过修改 `datasource.url` 即可切换。除默认的 SQLite 外，还支持 MySQL 和 PostgreSQL，满足高级用户的数据管理需求。

**SQLite（默认）：**

```yaml
datasource:
  url: "sqlite:/path/to/enote.db?mode=rwc"
  max-connections: 5
  min-connections: 1
```

- `mode=rwc`：读写模式，数据库文件不存在时自动创建。
- SQLite 为单文件数据库，无需安装额外服务，适合个人使用。

**MySQL：**

```yaml
datasource:
  url: "mysql://用户名:密码@服务器地址:3306/数据库名"
  max-connections: 20
  min-connections: 2
  connect_timeout: 10
  acquire-timeout: 5
  idle-time: 300
  max-lifetime: 1800
```

- 需预先安装 MySQL 5.7+ 或 MariaDB 10.3+ 并创建目标数据库。
- 示例：`url: "mysql://root:password@127.0.0.1:3306/enote"`

**PostgreSQL：**

```yaml
datasource:
  url: "postgres://用户名:密码@服务器地址:5432/数据库名"
  max-connections: 20
  min-connections: 2
  connect_timeout: 10
  acquire-timeout: 5
  idle-time: 300
  max-lifetime: 1800
```

- 需预先安装 PostgreSQL 12+ 并创建目标数据库。
- 示例：`url: "postgres://enote:password@127.0.0.1:5432/enote"`

> **注意：** 切换数据库类型后，原有数据库中的数据不会自动迁移。如需保留数据，请先使用导出功能备份笔记，切换后再导入。首次连接新数据库时，系统会自动创建所需的表结构。

### 21.5 Profile 配置管理

ENote 使用 Profile 模式管理多个数据库配置，配置文件存储在应用数据目录的 `profiles/` 子目录中。

**索引文件 `profiles.yml`：**

```yaml
active: local-sqlite    # 当前活跃的 Profile ID
auto-connect: false      # 是否自动连接上次使用的配置
```

**单个 Profile 配置文件示例（`profiles/local-sqlite.yml`）：**

```yaml
name: 本地笔记
icon: ''
datasource:
  type: sqlite
  path: /path/to/enote.db
security:
  content-encryption: false
```

**MySQL Profile 示例（`profiles/work-mysql.yml`）：**

```yaml
name: 工作数据库
icon: ''
datasource:
  type: mysql
  host: 192.168.1.100
  port: 3306
  database: enote
  username: enote_user
  auth-method: password
  ssl:
    mode: ''
security:
  content-encryption: true
```

> **注意：** 数据库密码和加密密钥不保存在配置文件中，而是安全存储在操作系统钥匙串中。

### 21.6 内容加密与密钥管理

启用内容加密后，系统使用 AES-256-GCM 算法对笔记内容进行加密保护。

**密钥存储：**

加密密钥存储在操作系统原生安全存储中：

| 操作系统 | 存储位置 | 管理方式 |
|----------|----------|----------|
| **macOS** | Keychain（钥匙串） | 可通过「钥匙串访问」应用查看 |
| **Windows** | Credential Store（凭据管理器） | 可通过「控制面板 > 凭据管理器」查看 |
| **Linux** | Secret Service（如 GNOME Keyring） | 可通过 `seahorse` 等工具查看 |

密钥条目使用应用标识符 `net.easycloudcn.enote` 作为服务名，格式为 `{profile_id}.encryption_key`。

**加密行为：**

- 笔记内容在保存到数据库前自动加密，读取时自动解密。
- 加密对用户完全透明，无需手动操作。
- 笔记标题**不加密**，仍可用于搜索。
- 笔记内容在数据库中以密文存储，即使直接打开数据库文件也无法读取原文。
- 笔记历史记录中的内容同样受加密保护。

### 21.7 SSL/TLS 证书认证

使用 MySQL 或 PostgreSQL 时，可通过 SSL/TLS 证书进行安全认证，保护数据传输安全。

**MySQL SSL 模式：**

| 模式 | 说明 |
|------|------|
| `disabled` | 不使用 SSL |
| `preferred` | 优先使用 SSL，不可用时回退 |
| `required` | 必须使用 SSL |
| `verify-ca` | 验证服务器 CA 证书 |
| `verify-identity` | 验证服务器身份（最严格） |

**PostgreSQL SSL 模式：**

| 模式 | 说明 |
|------|------|
| `disabled` | 不使用 SSL |
| `preferred` | 优先使用 SSL |
| `required` | 必须使用 SSL |
| `verify-ca` | 验证服务器 CA 证书 |
| `verify-full` | 完全验证（最严格） |

**所需证书文件（PEM 格式）：**

- **CA 证书：** 服务器 CA 根证书（`.pem` 或 `.crt`）
- **客户端证书：** 客户端身份证书（`.pem` 或 `.crt`）
- **客户端私钥：** 客户端私钥文件（`.pem` 或 `.key`）

证书文件路径在设置向导中通过文件选择器指定，保存在 Profile 配置文件中。

### 21.8 历史版本保护

系统自动记录笔记的每一次修改和删除操作的历史版本，包括操作前后的完整内容，支持随时追溯和对比，有效防止数据意外丢失。历史记录包含以下信息：

- 操作类型（新增、修改、删除）
- 操作来源（用户操作、AI 工具）
- 操作前的旧内容
- 操作后的新内容
- 操作发生时间
- 关联的笔记本和标签信息

## 23. 应用日志

ENote 提供完善的日志管理功能，支持数据库操作日志和文件系统日志两种类型，便于问题排查和操作审计。

### 23.1 打开日志管理

在设置对话框的"日志管理"区域，点击"打开日志管理"按钮，打开日志管理对话框。对话框包含"数据库日志"和"文件日志"两个选项卡。

### 23.2 数据库日志

数据库日志记录应用内的各类操作，包括笔记增删改、标签和笔记本操作、备份同步、加密操作等。

**日志字段：**

| 字段 | 说明 |
|------|------|
| 时间 | 操作发生的时间 |
| 级别 | DEBUG、INFO、WARN、ERROR |
| 模块 | 操作所属模块（notebook、note、tag、backup、sync、settings、encrypt、system、frontend） |
| 操作 | 具体执行的操作 |
| 消息 | 操作描述信息 |

**筛选与搜索：**
- **级别筛选：** 按日志级别过滤（全部/INFO/WARN/ERROR）。
- **模块筛选：** 按模块过滤。
- **关键词搜索：** 在消息内容中模糊搜索。

**管理操作：**
- **删除单条：** 点击日志右侧的删除按钮删除单条记录。
- **清空全部：** 清除所有数据库日志。
- **清理旧日志：** 删除 30 天前的旧日志。

日志级别以颜色标签区分：ERROR（红色）、WARN（黄色）、INFO（蓝色）、DEBUG（灰色）。列表支持分页浏览，每页 50 条。

### 23.3 文件日志

文件日志记录应用运行时的系统级日志，存储在应用数据目录的 `logs/` 子目录中。

**文件列表：**
- 左侧面板列出所有日志文件（`enote*.log`），显示文件大小和修改时间。
- 错误日志文件以红色标识。

**日志查看：**
- 选中文件后，右侧面板显示文件内容。
- 支持在文件内容中搜索，匹配文本以黄色高亮。
- ERROR 级别的行自动以红色高亮。

**管理操作：**
- **删除文件：** 删除选中的日志文件。
- **清理旧文件：** 删除 30 天前的旧日志文件。
- **刷新：** 重新加载文件列表。

### 23.4 前端日志级别

在设置对话框的"日志管理"区域，可设置前端日志级别：

| 级别 | 说明 |
|------|------|
| DEBUG | 记录所有调试信息 |
| INFO | 记录一般信息（默认） |
| WARN | 仅记录警告和错误 |
| ERROR | 仅记录错误 |
| NONE | 不记录前端日志 |

### 23.5 敏感数据保护

日志系统自动对敏感信息进行脱敏处理：

- **密码类字段：** 包含"password"、"密码"、"key"、"密钥"、"secret"、"token"等关键词的内容自动遮盖（前2字符 + **** + 后2字符）。
- **加密笔记内容：** 加密笔记的内容不会记录到日志中。

## 24. 自动更新

ENote 支持应用内自动更新，通过 GitHub Releases 分发新版本。更新包经过数字签名验证，确保安全可靠。

### 24.1 启动时自动检查

应用启动后会自动在后台检查是否有新版本可用：

- 启动 3 秒后静默检查，不影响正常使用
- 如果没有新版本，不会弹出任何提示
- 如果发现新版本，会弹出更新对话框
- 检查结果会记录在操作日志中（模块：`updater`）

### 24.2 手动检查更新

除了自动检查，还可以手动触发更新检查：

- **菜单栏**：点击 帮助 → 检查更新
- **命令面板**：按 `Ctrl+P`（macOS: `Cmd+P`），搜索「检查更新」

手动检查时：
- 如果已是最新版本，会弹出「当前已是最新版本」提示
- 如果检查失败（如网络问题），会弹出错误提示

### 24.3 下载与安装

发现新版本时，更新对话框会显示：

- 当前版本号和新版本号
- 更新说明（Release Notes）
- **立即更新** 和 **稍后提醒** 两个选项

点击「立即更新」后：

1. 显示下载进度条
2. 下载完成后自动安装
3. 安装完成后提示重启
4. 应用自动重启并加载新版本

> **注意**：下载过程中请勿关闭应用。如果下载失败，可以稍后再次尝试。
